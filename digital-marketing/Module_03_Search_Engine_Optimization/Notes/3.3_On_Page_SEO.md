# 3.3 On-Page SEO — Comprehensive Notes

---

## 3.3.1 Title Tags & Meta Descriptions

### Title Tags — The Single Most Important On-Page Element

**What it is:** The HTML `<title>` element that defines the page title. It appears in three critical places:
1. Browser tab
2. Search engine results page (SERP) as the clickable headline
3. Social media shares (if no OG title is set)

**Anatomy of a perfect title tag:**
```
Primary Keyword + Modifier + Brand
──────────────────────────────────────────────
Best Running Shoes for Flat Feet (2025 Guide) | Runner's World
     ↑                    ↑            ↑               ↑
 Primary keyword      Modifier    Freshness signal   Brand
```

**Technical specifications:**
| Spec | Recommendation |
|------|---------------|
| Length | 50-60 characters (Google truncates at ~580px) |
| Keyword placement | Front-load primary keyword (first 3-5 words) |
| Uniqueness | Every page must have a unique title |
| Brand inclusion | Append brand name at the end (separated by `|` or `-`) |
| Character encoding | Avoid special characters that may display incorrectly |

**Title tag formulas that work:**

| Formula | Example | Best For |
|---------|---------|----------|
| Primary Keyword: Secondary Benefit | "CRM Software: Boost Sales by 35% in 90 Days" | Product/landing pages |
| Number + Keyword + Year | "15 Best Email Marketing Tools (2025)" | Listicle/comparison posts |
| How to + Keyword + Outcome | "How to Write a Business Plan That Gets Funded" | How-to guides |
| Keyword + for + Audience | "Project Management for Remote Teams" | Niche targeting |
| Question format | "What Is SEO? Complete Beginner's Guide" | Informational content |

**CTR optimization modifiers (proven to increase click-through rate):**
- Power words: "Ultimate," "Complete," "Proven," "Free," "Essential"
- Numbers: "15 Best," "7 Steps," "101 Tips"
- Freshness: "(2025)," "(Updated)," "(New)"
- Brackets: "[Free Template]," "[Case Study]," "[Checklist]"
- Negative curiosity: "Mistakes," "Avoid," "Stop Doing This"

**Real-Time Example:**
- **Backlinko** tested title tag changes across 50 blog posts. Changing "SEO Tools" to "SEO Tools: The Complete List (59 Free & Paid Tools)" increased CTR from 4.2% to 7.8% — nearly doubling clicks without changing rankings. The addition of the number (59), parenthetical detail, and "Free & Paid" qualifier made the result more compelling in the SERP.

**Common mistakes to avoid:**
| Mistake | Why It's Bad | Fix |
|---------|-------------|-----|
| Duplicate titles across pages | Cannibalization, confuses Google | Unique title per page |
| Keyword stuffing | "Buy Shoes, Cheap Shoes, Best Shoes, Shoes Online" | One primary keyword, natural language |
| Too long (truncation) | User sees "Best Running Shoes for Flat Feet That Are Also Water..." | Keep under 60 characters |
| Missing keyword | "Welcome to Our Website" tells Google nothing | Include target keyword |
| All caps | LOOKS LIKE SPAM | Use Title Case or Sentence case |

**Testing title tags:**
1. Identify pages with high impressions but low CTR in Search Console
2. Write 2-3 title tag variations using the formulas above
3. Implement the new title and monitor CTR for 2-4 weeks
4. Keep the winner, iterate further

### Meta Descriptions

**What it is:** The HTML `<meta name="description">` tag that provides a summary of the page. Appears below the title in search results.

**Important truth:** Meta descriptions are NOT a ranking factor. Google confirmed this. However, they significantly impact CTR, which indirectly affects rankings (higher CTR → more traffic → engagement signals → better rankings).

**Specifications:**
| Spec | Recommendation |
|------|---------------|
| Length | 150-160 characters (Google truncates at ~920px) |
| Keyword inclusion | Include target keyword (Google bolds matching terms) |
| Call-to-action | End with a CTA: "Learn more," "Get started," "Shop now" |
| Uniqueness | Unique per page (or let Google auto-generate if low-priority) |

**Note:** Google rewrites meta descriptions 62% of the time, pulling content from the page instead. However, a well-written meta description is still worth implementing as Google uses it when it matches the query.

**Real-Time Example:**
- **Zapier's** meta description for "What is Automation": *"Automation uses technology to perform tasks with minimal human effort. Learn what automation means for your business and how to get started — with examples."* — This includes the keyword ("automation"), provides a clear definition, promises value ("examples"), and includes a subtle CTA ("learn" and "get started").

**Implementation Steps:**
1. Write meta descriptions for all landing pages and high-traffic blog posts
2. Include the primary keyword naturally
3. Summarize what the user will get/learn from the page
4. Include a compelling reason to click (unique data, free resource, actionable tips)
5. Monitor which pages Google is rewriting and optimize accordingly

## 3.3.2 Header Tags & Content Structure

### H1 Tag Best Practices

**What it is:** The main heading of a page, wrapped in `<h1>` tags. Think of it as the "headline" of your content.

**Rules:**
- One H1 per page (technically multiple are valid in HTML5, but one is the SEO best practice)
- Include primary keyword naturally
- Should closely match the title tag (but doesn't need to be identical)
- Should accurately describe the page content

**Real-Time Example:**
- **Page:** Ahrefs' backlink checker page
- **Title tag:** "Free Backlink Checker by Ahrefs: Check Backlinks to Any Site"
- **H1:** "Backlink Checker" (shorter, cleaner for on-page)
- The H1 matches user intent perfectly — they landed on a tool page, and the H1 confirms they're in the right place.

### H2-H6 Hierarchy

**Content structure should mirror an outline:**
```html
<h1>Complete Guide to Email Marketing</h1>              <!-- 1 per page -->
  <h2>What Is Email Marketing?</h2>                      <!-- Main sections -->
    <h3>History of Email Marketing</h3>                   <!-- Subsections -->
    <h3>Why Email Marketing Still Works</h3>
  <h2>Building Your Email List</h2>
    <h3>Lead Magnets That Convert</h3>
      <h4>eBooks and Whitepapers</h4>                     <!-- Sub-subsections -->
      <h4>Free Tools and Templates</h4>
    <h3>Opt-in Form Best Practices</h3>
  <h2>Email Marketing Tools Comparison</h2>
    <h3>Mailchimp</h3>
    <h3>ConvertKit</h3>
    <h3>ActiveCampaign</h3>
```

**Why proper hierarchy matters:**
- Helps Google understand content structure and relationships between sections
- Headers are weighted for featured snippet selection
- Improves accessibility (screen readers use header hierarchy)
- Increases readability and scanability (users scan headers before reading)

---

### Using Headers for Featured Snippet Optimization

**Paragraph snippet:** Place a concise 40-60 word answer immediately after the H2 that matches the query.

**List snippet:** Use H2 as the question, then H3s for each list item. Google pulls the H3s as the list.

**Table snippet:** Use an HTML table with a clear header row.

**Real-Time Example:**
- **Moz** targets featured snippets by structuring their headers as questions:
  - H2: "What Is Domain Authority?"
  - Immediately followed by a 50-word paragraph definition
  - This structure captured the featured snippet for "what is domain authority" (18K searches/mo)

---

### Content Length Optimization

**When long-form wins (1,500-5,000+ words):**
- Informational/educational queries ("complete guide to...")
- Competitive keywords where top results are comprehensive
- Topics requiring depth and nuance (YMYL content)
- Pillar pages designed to cover a topic exhaustively

**When short-form wins (300-800 words):**
- Transactional pages (product, pricing) — users want to buy, not read
- Simple answers ("what time zone is California in")
- Tool pages (calculators, converters)
- Local pages (location, hours, contact info)

**The truth about content length:** There's no universal ideal word count. The right length is however many words it takes to fully satisfy the user's intent — no more, no less.

**Real-Time Example:**
- **HubSpot** tested content length across 6,000 blog posts. They found posts between 2,100-2,400 words generated the most organic traffic on average. But their highest-traffic single page (Marketing Statistics) is 10,000+ words because the topic demands it. Meanwhile, their pricing page is 500 words because that's all you need.

## 3.3.3 URL Structure Optimization

### Clean, Descriptive URLs

**The ideal URL structure:**
```
Good:  example.com/blog/email-marketing-guide
Bad:   example.com/blog/p?id=42819&cat=3&ref=home
Ugly:  example.com/2024/01/15/the-ultimate-complete-best-comprehensive-guide-to-everything-about-email-marketing-for-beginners
```

**URL best practices:**

| Guideline | Example |
|-----------|---------|
| Use hyphens, not underscores | `/email-marketing` not `/email_marketing` |
| Lowercase only | `/about-us` not `/About-Us` |
| Short and descriptive | `/seo-guide` not `/the-complete-definitive-guide-to-search-engine-optimization` |
| Include keyword | `/best-crm-software` |
| Avoid parameters | `/shoes/nike-air-max` not `/products?id=239&color=red` |
| 3-5 words | Short enough to remember and share |
| Flat architecture | `/category/product` (2-3 levels deep maximum) |

### Handling URL Changes: 301 Redirects

**When to use 301 redirects:**
- Changing a URL (old URL → new URL)
- Merging pages (duplicate content consolidation)
- Moving to HTTPS (http:// → https://)
- Domain migration (old-domain.com → new-domain.com)
- Deleting a page that has backlinks (redirect to the next most relevant page)

**Implementation:**
```
# .htaccess (Apache)
Redirect 301 /old-url https://example.com/new-url

# nginx
server {
    rewrite ^/old-url$ /new-url permanent;
}
```

**Best Practices:**
- Never chain redirects (A → B → C → D). Each redirect should go directly to the final destination.
- Monitor redirect chains in Screaming Frog or Ahrefs Site Audit
- Update internal links to point to the new URL directly (don't rely on redirects for internal navigation)

---

### Canonical URLs

**What it is:** The `<link rel="canonical" href="...">` tag tells Google which URL is the "master" version when duplicate or similar content exists at multiple URLs.

**When to use canonicals:**
- URL parameters creating duplicates (`/shoes?color=red` → canonical to `/shoes`)
- HTTP and HTTPS versions
- www and non-www versions
- Syndicated content (point canonical to original)
- Paginated content (sometimes)

**Real-Time Example:**
- **Amazon** uses canonical tags extensively. A product can be accessed via: `/dp/B08XYZ123`, `/gp/product/B08XYZ123`, `/product/best-widget/dp/B08XYZ123`. All variations point canonical to the primary product URL, consolidating link equity.

## 3.3.4 Image & Media SEO

### Image Optimization Checklist

| Element | Best Practice | Example |
|---------|-------------|---------|
| **File name** | Descriptive, keyword-rich, hyphenated | `blue-running-shoes-nike-pegasus.webp` (not `IMG_2847.jpg`) |
| **Alt text** | Describe the image for accessibility + include keyword naturally | `alt="Nike Pegasus 41 blue running shoes on trail"` |
| **Format** | WebP (best compression/quality), AVIF (newer, better compression), PNG (transparency), SVG (icons/logos) | Use `<picture>` tag for format fallbacks |
| **Compression** | Reduce file size without visible quality loss | Tools: TinyPNG, ShortPixel, Squoosh |
| **Dimensions** | Serve images at the displayed size | Don't serve a 4000px image in a 400px container |
| **Lazy loading** | `loading="lazy"` on images below the fold | `<img src="shoe.webp" loading="lazy" alt="...">` |
| **Responsive** | Use `srcset` and `sizes` for different screen widths | Multiple image sizes for mobile, tablet, desktop |

**Real-Time Example:**
- **Pinterest** redesigned their image optimization pipeline and reduced average image size by 40% (switching from PNG/JPEG to WebP). This improved their LCP by 1.2 seconds, resulting in a 15% decrease in bounce rate and 10% increase in sign-ups. Image optimization alone moved the needle on their key business metric.

---

### Video SEO

**Optimization steps:**
1. Host on YouTube for maximum reach (or self-host if you want traffic to your site)
2. Create a transcript/text summary on the page alongside the video
3. Implement `VideoObject` schema markup
4. Create a video sitemap
5. Use descriptive thumbnails (custom, not auto-generated)
6. Include timestamps/chapters for key moments

**Schema markup example:**
```json
{
  "@context": "https://schema.org",
  "@type": "VideoObject",
  "name": "How to Make Cold Brew Coffee",
  "description": "Step-by-step guide to making cold brew at home",
  "thumbnailUrl": "https://example.com/thumbnail.jpg",
  "uploadDate": "2025-03-15",
  "duration": "PT5M30S",
  "contentUrl": "https://example.com/video.mp4"
}
```

## 3.3.5 Internal Linking Strategy

### Why Internal Links Matter

Internal links serve three critical functions:
1. **Crawlability:** Googlebot discovers pages by following links
2. **PageRank distribution:** Links pass authority from high-authority pages to important pages
3. **Context and relevance:** Anchor text tells Google what the linked page is about

```
Link Equity Flow Example:

Homepage (DR 70, most backlinks)
    │
    ├──► /services (receives link equity from homepage)
    │       │
    │       ├──► /services/seo (receives equity from services + homepage)
    │       └──► /services/ppc
    │
    ├──► /blog (receives link equity from homepage)
    │       │
    │       ├──► /blog/seo-guide (receives equity from blog + homepage)
    │       │       │
    │       │       └──► /services/seo (receives equity from blog post too!)
    │       │
    │       └──► /blog/ppc-guide
    │
    └──► /about
```

### Anchor Text Best Practices

| Type | Example | Usage |
|------|---------|-------|
| **Exact match** | "email marketing guide" | Use sparingly (10-20%) |
| **Partial match** | "our guide to email strategies" | Primary method (40-50%) |
| **Branded** | "read on HubSpot" | When linking to branded pages (10-15%) |
| **Natural/generic** | "learn more," "click here," "this article" | Acceptable but less informative (10-15%) |
| **URL** | "hubspot.com/blog/email" | Occasional use (5-10%) |

**Real-Time Example:**
- **Wikipedia** is the gold standard of internal linking. Every article contains dozens of contextual internal links using descriptive anchor text. A page about "World War II" links to "Adolf Hitler," "Allied powers," "Pearl Harbor" — each link uses the exact topic name as anchor text, helping Google understand every page's subject.

### Internal Link Audit Process

**Steps:**
1. Crawl your site with Screaming Frog
2. Export the internal link report
3. Identify orphan pages (pages with zero internal links pointing to them)
4. Identify top pages by backlinks that aren't linking internally to important pages
5. Identify pages with too many links (diminishing returns past 100+)
6. Add contextual internal links to high-authority pages pointing to conversion pages

**Tools:**
| Tool | Feature |
|------|---------|
| Screaming Frog | Internal link visualization, orphan page detection |
| Ahrefs → Site Audit | Internal link opportunities, orphan pages |
| Link Whisper (WordPress) | AI-powered internal link suggestions |
| Yoast SEO (WordPress) | Internal linking suggestions |
| Google Search Console → Links | Top internally linked pages |

## 3.3.6 Content Optimization

### Keyword Density — The Modern View

**Old school:** Aim for 2-3% keyword density (mentioning "running shoes" 20 times in a 1,000-word article).

**Modern approach:** There's no magic density number. Include your keyword naturally in:
- Title tag
- H1
- First 100 words
- 2-3 times naturally in the body
- One H2 heading
- Alt text of one relevant image
- URL

**The real optimization:** Focus on topical completeness (covering all aspects of the topic) rather than repeating one keyword.

---

### Content Freshness and Update Strategy

**Why freshness matters:** Google gives a "freshness boost" to recently updated content, especially for queries that demand current information (news, trends, annual guides).

**Content update strategy:**

| Action | When | How |
|--------|------|-----|
| **Full rewrite** | Content is outdated or fundamentally wrong | Rewrite from scratch, keep URL |
| **Major update** | Core info is good but needs new sections, data | Add new sections, update stats, refresh screenshots |
| **Minor refresh** | Content is solid but dates are stale | Update year references, add latest examples |
| **No-touch** | Evergreen content performing well | Monitor rankings — don't fix what isn't broken |

**Implementation:**
1. Quarterly: Review top 50 pages by traffic — are they still accurate?
2. Monthly: Check Search Console for declining positions (position dropped 5+ spots)
3. Annually: Update all "best of [year]" and dated content
4. Show the update: Display "Last updated: [date]" visibly on the page

**Real-Time Example:**
- **Backlinko** refreshes their "Google Ranking Factors" post annually. Each update adds new factors, removes deprecated ones, updates data points, and adds fresh examples. The URL stays the same, preserving 15,000+ backlinks. Each refresh triggers a ranking boost, and the post consistently ranks #1 for "Google ranking factors" — a keyword worth an estimated $200K/year in organic traffic value.

---

### Content Pruning: When to Update, Merge, or Delete

**Decision framework:**
```
Page has organic traffic?
├── YES → Don't delete. Update or leave it.
│
├── NO → Does it have backlinks?
│         ├── YES → Redirect to a better page (301)
│         └── NO → Does it serve a business purpose?
│                   ├── YES → Improve quality and promote
│                   └── NO → Delete (or noindex)
```

**Real-Time Example:**
- **Ahrefs** audited their blog (900+ posts) and found 266 posts generating zero organic traffic and no backlinks. They deleted 49, merged 62 into stronger existing posts (with 301 redirects), and updated 155. Result: overall organic traffic increased by 7.1% within 3 months. Removing dead weight improved their site's quality score and let Google focus on their best content.

---

### NLP Optimization for Search Algorithms

**What it is:** Structuring content so that NLP models (BERT, MUM) can easily extract meaning, entities, and relationships from your content.

**Tactics:**
1. **Clear definitions:** Define terms before using them
2. **Entity-rich content:** Mention specific people, places, products, and concepts
3. **Question-answer format:** Match how NLP models parse Q&A pairs
4. **Concise paragraphs:** 2-3 sentences per paragraph (helps with snippet extraction)
5. **Semantic HTML:** Use proper headings, lists, tables — not just visual formatting
6. **Structured data:** Reinforce content meaning with JSON-LD schema

## On-Page SEO Implementation Checklist

```
TITLE & META
□ Unique title tag per page (50-60 chars, keyword front-loaded)
□ Unique meta description per page (150-160 chars, includes CTA)
□ No duplicate titles/descriptions across the site

HEADERS & STRUCTURE
□ One H1 per page containing primary keyword
□ Logical H2-H4 hierarchy (no skipped levels)
□ Headers contain secondary keywords naturally
□ Table of contents for posts >1,500 words

URLs
□ Clean, hyphenated, lowercase URLs
□ URLs contain primary keyword
□ 3 folder levels maximum
□ Canonical tags on all pages
□ Proper 301 redirects for changed URLs

IMAGES & MEDIA
□ Descriptive file names (not IMG_2847.jpg)
□ Alt text on all images (descriptive, keyword-aware)
□ WebP format with fallbacks
□ Compressed to <200KB per image (where possible)
□ Lazy loading on below-the-fold images
□ Video schema markup on pages with video

INTERNAL LINKS
□ 3-5 contextual internal links per page
□ Descriptive anchor text (not "click here")
□ Key pages accessible within 3 clicks from homepage
□ No orphan pages
□ Pillar-cluster linking model implemented

CONTENT
□ Primary keyword in first 100 words
□ Semantic/related keywords included naturally
□ Content length matches intent and competition
□ Content is fresh and regularly updated
□ Above-the-fold content adds immediate value
□ NLP-friendly: clear definitions, entities, structured format
```
