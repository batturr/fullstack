# TypeScript 5.9 MCQ - Set 1 (Beginner Level)

**1. What is TypeScript primarily described as?**

a) A completely different runtime language that replaces the JavaScript engine
b) A typed superset of JavaScript that compiles to plain JavaScript
c) A browser-only scripting dialect with no build step
d) A database query language for JSON APIs

**Answer: b) A typed superset of JavaScript that compiles to plain JavaScript**

---

**2. Which statement best explains a common reason teams adopt TypeScript?**

a) It removes the need for any JavaScript knowledge
b) It catches many errors at compile time and improves editor tooling via static types
c) It guarantees zero runtime errors in production
d) It only runs in Deno and cannot be used with Node.js

**Answer: b) It catches many errors at compile time and improves editor tooling via static types**

---

**3. Compared to plain JavaScript, which is true about TypeScript at runtime in typical setups?**

a) Browsers execute `.ts` files directly without transformation
b) TypeScript types are erased during compilation; runtime behavior matches the emitted JS
c) TypeScript adds a global type checker that runs inside every browser tab
d) `any` types are preserved as runtime validation wrappers

**Answer: b) TypeScript types are erased during compilation; runtime behavior matches the emitted JS**

---

**4. Which command installs TypeScript 5.9 as a dev dependency in an npm project?**

b) `npm install -D typescript`
