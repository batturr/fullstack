## What are Conditional Statements?

**Conditional statements** control the flow of execution in a program. They evaluate a condition and decide which block of code to execute based on the result.

### Key Points:
- Conditions evaluate to **boolean** values (`true` or `false`)
- JavaScript supports multiple types of conditional statements
- Conditions can use comparison operators (`==`, `===`, `<`, `>`, etc.)
- Conditions can use logical operators (`&&`, `||`, `!`)
- Truthy and falsy values affect conditions

## Types of Conditional Statements

JavaScript supports **5 types** of conditional statements:

| # | Statement | Description | Use Case |
|---|-----------|-------------|----------|
| 1 | `if` | Executes code if condition is true | Single condition check |
| 2 | `if...else` | Executes one block or another | Two-way decision |
| 3 | `nested if` | if inside another if | Multi-level conditions |
| 4 | `if...else if...else` | Multiple condition checks | Multiple options |
| 5 | `switch` | Matches expression against cases | Fixed value matching |

## if Statement

The `if` statement executes a block of code **only if** the specified condition is `true`. If the condition is `false`, the code inside the block is skipped.

### Syntax:
```javascript
if (condition) {
    // block of code to be executed if the condition is true
}
```

> **Note:** Without curly braces `{}`, only the first statement after `if` is treated as part of the if block.

### Example 1: Basic if
```javascript
let age = 19;

if (age > 18) {
    console.log("Congratulations, You are eligible to drive");
}
// Output: Congratulations, You are eligible to drive
```

### Example 2: if with Multiple Statements
```javascript
let temperature = 35;

if (temperature > 30) {
    console.log("It's hot outside!");
    console.log("Stay hydrated!");
    console.log("Use sunscreen!");
}
// Output:
// It's hot outside!
// Stay hydrated!
// Use sunscreen!
```

### Example 3: if Without Curly Braces
```javascript
// Only the FIRST statement is part of if block
let x = 10;

if (x > 5)
    console.log("This is inside if");     // Part of if
    console.log("This always executes");  // NOT part of if (always runs)

// Output:
// This is inside if
// This always executes

// ‚ö†Ô∏è WARNING: Always use curly braces to avoid confusion!
```

### Example 4: if with Logical Operators
```javascript
let age = 25;
let hasLicense = true;

// AND (&&) ‚Äî both conditions must be true
if (age >= 18 && hasLicense) {
    console.log("You can drive!");
}
// Output: You can drive!

// OR (||) ‚Äî at least one condition must be true
let isWeekend = false;
let isHoliday = true;

if (isWeekend || isHoliday) {
    console.log("No work today!");
}
// Output: No work today!

// NOT (!) ‚Äî inverts the condition
let isLoggedIn = false;

if (!isLoggedIn) {
    console.log("Please log in first");
}
// Output: Please log in first
```

### Example 5: if with Different Conditions
```javascript
let num = 15;

// Comparison operators
if (num > 10) console.log("Greater than 10");    // ‚úÖ
if (num < 20) console.log("Less than 20");       // ‚úÖ
if (num >= 15) console.log("Greater or equal 15"); // ‚úÖ
if (num <= 15) console.log("Less or equal 15");   // ‚úÖ
if (num === 15) console.log("Equal to 15");       // ‚úÖ
if (num !== 10) console.log("Not equal to 10");   // ‚úÖ
```

## if...else Statement

The `if...else` statement executes one block of code when the condition is `true` and another block when the condition is `false`.

### Syntax:
```javascript
if (condition) {
    // block of code if condition is TRUE
} else {
    // block of code if condition is FALSE
}
```

### Example 1: Basic if...else
```javascript
let i = 10;

if (i < 15) {
    console.log("i is less than 15");
} else {
    console.log("i is greater than or equal to 15");
}
// Output: i is less than 15
```

### Example 2: Even or Odd Check
```javascript
let number = 7;

if (number % 2 === 0) {
    console.log(`${number} is Even`);
} else {
    console.log(`${number} is Odd`);
}
// Output: 7 is Odd
```

### Example 3: Login Validation
```javascript
let username = "admin";
let password = "12345";

if (username === "admin" && password === "12345") {
    console.log("Login successful! Welcome, Admin.");
} else {
    console.log("Invalid username or password.");
}
// Output: Login successful! Welcome, Admin.
```

### Example 4: Ternary Operator (Shorthand if...else)
```javascript
// Regular if...else
let age = 20;
let status;

if (age >= 18) {
    status = "Adult";
} else {
    status = "Minor";
}

// Same thing using ternary operator
let status2 = age >= 18 ? "Adult" : "Minor";

console.log(status);   // "Adult"
console.log(status2);  // "Adult"
```

### Example 5: Positive, Negative, or Zero
```javascript
let num = -5;

if (num > 0) {
    console.log(`${num} is Positive`);
} else if (num < 0) {
    console.log(`${num} is Negative`);
} else {
    console.log(`${num} is Zero`);
}
// Output: -5 is Negative
```

## Nested if Statement

A **nested if** is an `if` statement placed inside another `if` statement. It allows you to check multiple conditions in a hierarchical manner.

### Syntax:
```javascript
if (condition1) {
    // Executes when condition1 is true
    if (condition2) {
        // Executes when both condition1 AND condition2 are true
    }
}
```

### Example 1: Basic Nested if
```javascript
let i = 10;

if (i === 10) {
    if (i < 15) {
        console.log("i is smaller than 15");

        if (i < 12) {
            console.log("i is smaller than 12 too");
        } else {
            console.log("i is greater than or equal to 12");
        }
    }
}
// Output:
// i is smaller than 15
// i is smaller than 12 too
```

### Example 2: Multiple Level Nesting
```javascript
let age = 25;
let hasID = true;
let hasMembership = true;

if (age >= 18) {
    console.log("Age verified ‚úÖ");

    if (hasID) {
        console.log("ID verified ‚úÖ");

        if (hasMembership) {
            console.log("Membership verified ‚úÖ");
            console.log("Access Granted!");
        } else {
            console.log("No membership. Access Denied.");
        }
    } else {
        console.log("No ID. Access Denied.");
    }
} else {
    console.log("Must be 18 or older. Access Denied.");
}
// Output:
// Age verified ‚úÖ
// ID verified ‚úÖ
// Membership verified ‚úÖ
// Access Granted!
```

### Example 3: Nested if with else
```javascript
let num = 15;

if (num > 0) {
    if (num % 2 === 0) {
        console.log(`${num} is a Positive Even number`);
    } else {
        console.log(`${num} is a Positive Odd number`);
    }
} else if (num < 0) {
    if (num % 2 === 0) {
        console.log(`${num} is a Negative Even number`);
    } else {
        console.log(`${num} is a Negative Odd number`);
    }
} else {
    console.log("The number is Zero");
}
// Output: 15 is a Positive Odd number
```

### Example 4: Flatten Nested if with Logical Operators
```javascript
// Nested version (harder to read)
let age = 25;
let hasID = true;

if (age >= 18) {
    if (hasID) {
        console.log("Entry allowed");
    }
}

// Flattened version (cleaner ‚Äî preferred)
if (age >= 18 && hasID) {
    console.log("Entry allowed");
}
```

## if...else if...else Statement (Ladder)

The `if...else if...else` statement checks **multiple conditions** sequentially. As soon as one condition is `true`, its block executes and the rest are skipped. If none are `true`, the `else` block runs.

### Syntax:
```javascript
if (condition1) {
    // executes if condition1 is true
} else if (condition2) {
    // executes if condition1 is false AND condition2 is true
} else if (condition3) {
    // executes if condition1 & condition2 are false AND condition3 is true
} else {
    // executes if ALL conditions are false
}
```

### Example 1: Basic if...else if
```javascript
let i = 20;

if (i === 10) {
    console.log("i is 10");
} else if (i === 15) {
    console.log("i is 15");
} else if (i === 20) {
    console.log("i is 20");
} else {
    console.log("i is not present");
}
// Output: i is 20
```

### Example 2: Grade Calculator
```javascript
let marks = 78;
let grade;

if (marks >= 90) {
    grade = "A+ (Outstanding)";
} else if (marks >= 80) {
    grade = "A (Excellent)";
} else if (marks >= 70) {
    grade = "B (Good)";
} else if (marks >= 60) {
    grade = "C (Average)";
} else if (marks >= 50) {
    grade = "D (Below Average)";
} else if (marks >= 35) {
    grade = "E (Pass)";
} else {
    grade = "F (Fail)";
}

console.log(`Marks: ${marks}, Grade: ${grade}`);
// Output: Marks: 78, Grade: B (Good)
```

### Example 3: BMI Calculator
```javascript
let weight = 70;  // kg
let height = 1.75; // meters
let bmi = weight / (height * height);

let category;

if (bmi < 18.5) {

console.log(`BMI: ${bmi.toFixed(1)}, Category: ${category}`);
```

### Example 4: Season Finder

```javascript
let month = 3; // March

let season;

if (month >= 3 && month <= 5) {
```

## switch Statement

The `switch` statement evaluates an expression and matches it against multiple `case` values. It's an alternative to long `if...else if` chains when comparing a single value against multiple options.

### Syntax:
```javascript
switch (expression) {
    case value1:
        // code block 1
        break;
    case value2:
        // code block 2
        break;
    case value3:
        // code block 3
        break;
    default:
        // default code block (if no case matches)
}
```

### Key Points:
- Uses **strict comparison** (`===`) to match cases
- The `break` keyword exits the switch block
- Without `break`, execution **falls through** to the next case
- `default` is optional and runs when no case matches
- `default` can be placed anywhere (not just at the end)

### Example 1: Basic switch
```javascript
let day = 2;
let dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    case 4:
        dayName = "Thursday";
        break;
    case 5:
        dayName = "Friday";
        break;
    case 6:
        dayName = "Saturday";
        break;
    case 7:
        dayName = "Sunday";
        break;
    default:
        dayName = "Invalid day";
}

console.log(dayName);
// Output: Tuesday
```

### Example 2: Grade with switch
```javascript
let grade = 'B';
let result;

switch (grade) {
    case 'A':
        result = "A (Excellent)";
        break;
    case 'B':
        result = "B (Average)";
        break;
    case 'C':
        result = "C (Below Average)";
        break;
    case 'D':
        result = "D (Poor)";
        break;
    case 'F':
        result = "F (Fail)";
        break;
    default:
        result = "Invalid Grade";
}

console.log(result);
// Output: B (Average)
```

### Example 3: Default at Top
```javascript
let day = 8;
let dayName;

switch (day) {
    default:
        dayName = "Invalid day";
        break;
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
}

console.log(dayName);
// Output: Invalid day
```

### Example 4: Grouping Cases (Fall-Through)
```javascript
// Multiple cases can share the same code block
let day = 6;
let type;

switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        type = "Weekday";
        break;
    case 6:
    case 7:
        type = "Weekend";
        break;
    default:
        type = "Invalid day";
}

console.log(type);
// Output: Weekend
```

### Example 5: switch with Expressions
```javascript
let score = 85;

switch (true) {
    case (score >= 90):
        console.log("Grade: A");
        break;
    case (score >= 80):
        console.log("Grade: B");
        break;
    case (score >= 70):
        console.log("Grade: C");
        break;
    case (score >= 60):
        console.log("Grade: D");
        break;
    default:
        console.log("Grade: F");
}
// Output: Grade: B
```

### Example 6: switch vs if...else if
```javascript
// Using if...else if
let fruit = "apple";

if (fruit === "apple") {
    console.log("üçé Apple");
} else if (fruit === "banana") {
    console.log("üçå Banana");

    console.log("üçä Orange");

    console.log("Unknown fruit");

    case "banana":
        console.log("üçå Banana");
        break;

        console.log("üçä Orange");

        console.log("Unknown fruit");

```

### Example 7: Without break (Fall-Through Behavior)
```javascript

        console.log("One");

// Two

// (All three run because of missing break statements!)
```

### Example 8: switch with return in Functions

## Truthy and Falsy in Conditions

JavaScript uses **truthy** and **falsy** values in conditional checks. Any value can be used as a condition.

### Falsy Values (evaluate to false):
```javascript
if (false)      { }  // false
if (0)          { }  // false
if (-0)         { }  // false
if (0n)         { }  // false (BigInt zero)
if ("")         { }  // false (empty string)
if (null)       { }  // false
if (undefined)  { }  // false
if (NaN)        { }  // false
```

### Truthy Values (evaluate to true):
```javascript
if (true)       { }  // true
if (1)          { }  // true
if (-1)         { }  // true
if ("hello")    { }  // true
if ("0")        { }  // true (non-empty string!)
if ("false")    { }  // true (non-empty string!)
if ([])         { }  // true (empty array!)
if ({})         { }  // true (empty object!)
if (function(){}) { } // true
```

### Example 1: Truthy/Falsy in Practice
```javascript
let username = "";

if (username) {
    console.log(`Welcome, ${username}`);
} else {
    console.log("No username provided");
}
// Output: No username provided

username = "Alice";

if (username) {
    console.log(`Welcome, ${username}`);
}
// Output: Welcome, Alice
```

### Example 2: Checking for null/undefined
```javascript
let data = null;

if (data) {
    console.log("Data exists:", data);
} else {
    console.log("No data available");
}
// Output: No data available
```

## Short-Circuit Evaluation

Logical operators can be used as shorthand for conditional statements.

### Example 1: AND (&&) ‚Äî Execute if Truthy
```javascript
let isLoggedIn = true;

// Instead of: if (isLoggedIn) { showDashboard(); }
isLoggedIn && console.log("Dashboard displayed");
// Output: Dashboard displayed

let isAdmin = false;
isAdmin && console.log("Admin panel");
// Nothing happens (isAdmin is false)
```

### Example 2: OR (||) ‚Äî Default Values
```javascript
let name = "" || "Guest";
console.log(name);  // "Guest" (empty string is falsy)

let count = 0 || 10;
console.log(count);  // 10 (0 is falsy)

let value = null || "Default";
console.log(value);  // "Default"
```

### Example 3: Double NOT (!!) ‚Äî Convert to Boolean
```javascript
console.log(!!"hello");  // true
console.log(!!"");        // false
console.log(!!0);         // false
console.log(!!42);        // true
console.log(!!null);      // false
```

## Nullish Coalescing (??) and Optional Chaining (?.) in Conditions

### Nullish Coalescing (??)
Returns the right-hand value only when the left is `null` or `undefined` (not for `0`, `""`, or `false`).

```javascript
let count = 0;

// Using || (treats 0 as falsy ‚Äî wrong!)
let result1 = count || 10;
console.log(result1);  // 10

// Using ?? (treats only null/undefined as missing ‚Äî correct!)
let result2 = count ?? 10;
console.log(result2);  // 0
```

### Optional Chaining (?.)
Safely accesses nested properties without errors.

```javascript
let user = {
    name: "John",
    address: {
        city: "New York"
    }
};

// Without optional chaining (can throw error)
// console.log(user.phone.number);  // TypeError!

// With optional chaining (returns undefined safely)
console.log(user.phone?.number);  // undefined

// Combining with ??
let city = user.address?.city ?? "Unknown";
console.log(city);  // "New York"

let country = user.address?.country ?? "Unknown";
console.log(country);  // "Unknown"
```

## Best Practices

### ‚úÖ Do:

1. **Always Use Curly Braces `{}`**
   ```javascript
   // Good
   if (condition) {
       doSomething();
   }

   // Avoid (error-prone)
   if (condition)
       doSomething();
   ```

2. **Use `===` (Strict Equality) in Conditions**
   ```javascript
   // Good
   if (value === 5) { }

   // Avoid
   if (value == 5) { }  // Type coercion can cause bugs
   ```

3. **Use Early Returns to Reduce Nesting**
   ```javascript
   // Good ‚Äî early return
   function processUser(user) {
       if (!user) return "No user";
       if (!user.active) return "User inactive";
       return `Welcome, ${user.name}`;
   }

   // Avoid ‚Äî deep nesting
   function processUser2(user) {
       if (user) {
           if (user.active) {
               return `Welcome, ${user.name}`;
           } else {
               return "User inactive";
           }
       } else {
           return "No user";
       }
   }
   ```

4. **Use switch for Multiple Fixed Values**
   ```javascript
   // Good ‚Äî switch for fixed values
   switch (status) {
       case "active": return "üü¢ Active";
       case "inactive": return "üî¥ Inactive";
       case "pending": return "üü° Pending";
       default: return "Unknown";
   }
   ```

5. **Use Ternary for Simple Assignments**
   ```javascript
   // Good ‚Äî simple ternary
   let status = age >= 18 ? "Adult" : "Minor";

   // Avoid ‚Äî complex ternary
   // let x = a ? b ? c : d : e ? f : g;
   ```

6. **Flatten Nested Conditions with && Operator**
   ```javascript
   // Good
   if (age >= 18 && hasID && hasMembership) {
       grantAccess();
   }

   // Avoid
   if (age >= 18) {
       if (hasID) {
           if (hasMembership) {
               grantAccess();
           }
       }
   }
   ```

7. **Use `??` Instead of `||` for Default Values**
   ```javascript
   // Good
   let count = userCount ?? 0;   // Only null/undefined triggers default

   // Risky
   let count2 = userCount || 0;  // 0 would also trigger default!
   ```

## Common Mistakes to Avoid

### ‚ùå Don't:

1. **Don't Use `=` Instead of `===`**
   ```javascript
   let x = 10;

   // BAD ‚Äî assignment, not comparison!
   if (x = 5) {
       console.log(x);  // 5 (always true because assignment returns 5)
   }

   // GOOD ‚Äî comparison
   if (x === 5) {
       console.log(x);
   }
   ```

2. **Don't Forget break in switch**
   ```javascript
   // BAD ‚Äî missing break causes fall-through
   switch (value) {
       case 1:
           console.log("One");
           // Falls through to case 2!
       case 2:
           console.log("Two");
           break;
   }

   // GOOD
   switch (value) {
       case 1:
           console.log("One");
           break;
       case 2:
           console.log("Two");
           break;
   }
   ```

3. **Don't Use Too Many Nested ifs**
   ```javascript
   // BAD ‚Äî deeply nested
   if (a) {
       if (b) {
           if (c) {
               if (d) {
                   doSomething();
               }
           }
       }
   }

   // GOOD ‚Äî flat with early returns or combined conditions
   if (a && b && c && d) {
       doSomething();
   }
   ```

4. **Don't Compare Booleans to true/false**
   ```javascript
   let isActive = true;

   // BAD ‚Äî redundant
   if (isActive === true) { }
   if (isActive == true) { }

   // GOOD ‚Äî direct boolean check
   if (isActive) { }
   if (!isActive) { }
   ```

5. **Don't Use switch Without default**
   ```javascript
   // BAD ‚Äî no fallback
   switch (color) {
       case "red": break;
       case "blue": break;
   }

   // GOOD ‚Äî always include default
   switch (color) {
       case "red": break;
       case "blue": break;
       default: console.log("Unknown color");
   }
   ```

6. **Don't Forget That switch Uses Strict Comparison**
   ```javascript
   let num = "5";

   switch (num) {
       case 5:
           console.log("Number 5");
           break;
       case "5":
           console.log("String '5'");  // This matches! (=== comparison)
           break;
   }
   // Output: String '5'
   ```

## Comparison Table

### When to Use Which Statement:

| Statement | Best For | Example |
|-----------|----------|---------|
| `if` | Single condition | Check if user is logged in |
| `if...else` | Two-way decision | Login success or failure |
| `nested if` | Hierarchical checks | Multi-level validation |
| `if...else if` | Multiple ranges/conditions | Grade calculator, BMI |
| `switch` | Fixed value matching | Day names, menu options |
| Ternary `? :` | Simple inline assignment | `status = age >= 18 ? "Adult" : "Minor"` |
| `&&` short-circuit | Conditional execution | `isLoggedIn && showDashboard()` |
| `\|\|` short-circuit | Default values | `name = input \|\| "Guest"` |
| `??` nullish | Safe defaults (keep 0, "") | `count = input ?? 0` |

### if...else if vs switch:

| Feature | if...else if | switch |
|---------|-------------|--------|
| **Comparison** | Any condition (ranges, expressions) | Strict equality (`===`) only |
| **Flexibility** | Can use `&&`, `\|\|`, `<`, `>`, etc. | Single value matching |
| **Readability** | Good for few conditions | Better for many fixed values |
| **Performance** | Checks sequentially | Can be optimized by engine |
| **Fall-through** | No | Yes (without `break`) |
| **Best for** | Ranges, complex conditions | Menu options, fixed values |

## Summary

### Key Takeaways:

1. **`if`** ‚Äî executes code when condition is `true`
2. **`if...else`** ‚Äî two-way decision (true or false)
3. **`nested if`** ‚Äî hierarchical condition checks (avoid deep nesting)
4. **`if...else if...else`** ‚Äî multiple conditions checked sequentially
5. **`switch`** ‚Äî matches a single value against multiple cases
6. Always use **`===`** in conditions (strict equality)
7. Always add **`break`** in switch cases (or use `return` in functions)
8. Use **early returns** to avoid deep nesting
9. Use **ternary operator** for simple inline assignments
10. Use **`??`** instead of `||` for safe default values

---

**Remember**: Write clean, readable conditions! Prefer flat logic over deep nesting, and choose the right statement type for the right situation.
