# JavaScript Reserved Words

Reserved words in JavaScript are keywords that have special meaning and functionality in the language. They cannot be used as identifiers for variables, functions, methods, loop labels, or object names.

> **Important**: All reserved words are in lowercase only.

## Control Flow

Keywords used to control the flow of execution in JavaScript programs.

| Keyword   | Description                                        | Example |
|-----------|----------------------------------------------------|---------
| `if`      | Conditional statement                              | `if (x > 0) { ... }` |
| `else`    | Alternative branch for if statement                | `if (x > 0) { ... } else { ... }` |
| `switch`  | Multi-way branch statement                         | `switch(value) { ... }` |
| `case`    | Label in a switch statement                        | `case 1: ...` |
| `default` | Default case in switch statement                   | `default: ...` |

### Example
```javascript
let score = 85;

if (score >= 90) {
  console.log("A grade");
} else if (score >= 80) {
  console.log("B grade");
} else {
  console.log("C grade");
}

switch (score) {
  case 100:
    console.log("Perfect!");
    break;
  case 90:
    console.log("Excellent!");
    break;
  default:
    console.log("Good effort!");
}
```

## Loops

Keywords used for iterating and controlling loop execution.

| Keyword    | Description                                      | Example |
|------------|--------------------------------------------------|---------|
| `for`      | Creates a loop                                   | `for (let i = 0; i < 10; i++) { ... }` |
| `while`    | Creates a loop that executes while condition is true | `while (x > 0) { ... }` |
| `do`       | Executes a block of code once, then repeats while condition is true | `do { ... } while (x > 0)` |
| `break`    | Terminates a loop or switch statement            | `break;` |
| `continue` | Skips the current iteration of a loop            | `continue;` |

### Example
```javascript
// For loop
for (let i = 0; i < 5; i++) {
  console.log(i);
}

// While loop
let count = 0;
while (count < 5) {
  console.log(count);
  count++;
}

// Do-while loop
let num = 0;
do {
  console.log(num);
  num++;
} while (num < 5);

// Break and continue
for (let i = 0; i < 10; i++) {
  if (i === 5) break;        // Stop loop at 5
  if (i % 2 === 0) continue; // Skip even numbers
  console.log(i);
}
```

## Declarations

Keywords used to declare variables with different scopes and mutability.

| Keyword | Description                                      | Scope          | Reassignable | Hoisted |
|---------|--------------------------------------------------|----------------|--------------|---------|
| `var`   | Declares a variable (old style)                  | Function       | Yes          | Yes     |
| `let`   | Declares a block-scoped variable                 | Block          | Yes          | No      |
| `const` | Declares a block-scoped constant                 | Block          | No           | No      |

### Example
```javascript
// var - function scoped
var x = 10;
x = 20; // Can be reassigned

// let - block scoped
let y = 30;
y = 40; // Can be reassigned

// const - block scoped, cannot be reassigned
const z = 50;
// z = 60; // Error: Assignment to constant variable

// Block scope demonstration
{
  let blockVar = 100;
  const blockConst = 200;
}
// console.log(blockVar); // Error: blockVar is not defined
```

## Functions and Classes

Keywords related to function and class declarations.

| Keyword       | Description                                    | Example |
|---------------|------------------------------------------------|---------|
| `function`    | Declares a function                            | `function myFunc() { ... }` |
| `return`      | Exits a function and returns a value           | `return value;` |
| `class`       | Declares a class                               | `class MyClass { ... }` |
| `extends`     | Creates a class inheritance                    | `class Child extends Parent { ... }` |
| `constructor` | Method for creating and initializing objects   | `constructor() { ... }` |
| `super`       | Calls parent class constructor or methods      | `super();` |

### Example
```javascript
// Function
function greet(name) {
  return `Hello, ${name}!`;
}

// Class
class Animal {
  constructor(name) {
    this.name = name;
  }
  
  speak() {
    return `${this.name} makes a sound.`;
  }
}

// Class with inheritance
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // Call parent constructor
    this.breed = breed;
  }
  
  speak() {
    return `${this.name} barks!`;
  }
}

const dog = new Dog("Rex", "German Shepherd");
console.log(dog.speak()); // Rex barks!
```

## Scope and Context

Keywords related to object context and instantiation.

| Keyword | Description                                      | Example |
|---------|--------------------------------------------------|---------|
| `this`  | Refers to the current object context             | `this.property` |
| `new`   | Creates an instance of an object                 | `new MyClass()` |

### Example
```javascript
class User {
  constructor(name) {
    this.name = name; // 'this' refers to the current instance
  }
  
  greet() {
    console.log(`Hello, I'm ${this.name}`);
  }
}

const user = new User("Alice"); // 'new' creates an instance
user.greet(); // Hello, I'm Alice
```

## Operators

Keywords that perform operations on values and variables.

| Keyword      | Description                                    | Example |
|--------------|------------------------------------------------|---------|
| `typeof`     | Returns the type of a variable                 | `typeof x` |
| `instanceof` | Tests if an object is an instance of a class   | `obj instanceof Class` |
| `delete`     | Removes a property from an object              | `delete obj.prop` |
| `in`         | Checks if property exists in an object         | `'prop' in obj` |
| `void`       | Evaluates an expression and returns undefined  | `void(0)` |

### Example
```javascript
// typeof
console.log(typeof 42);        // "number"
console.log(typeof "hello");   // "string"
console.log(typeof true);      // "boolean"
console.log(typeof {});        // "object"

// instanceof
class Car {}
const myCar = new Car();
console.log(myCar instanceof Car); // true

// delete
const obj = { name: "John", age: 30 };
delete obj.age;
console.log(obj); // { name: "John" }

// in
console.log('name' in obj); // true
console.log('age' in obj);  // false

// void
void function() {
  console.log("This returns undefined");
}();
```

## Exceptions

Keywords for error handling and exception management.

| Keyword   | Description                                    | Example |
|-----------|------------------------------------------------|---------|
| `try`     | Defines a block to test for errors             | `try { ... }` |
| `catch`   | Handles errors from try block                  | `catch(error) { ... }` |
| `finally` | Executes code after try/catch regardless       | `finally { ... }` |
| `throw`   | Throws a custom error                          | `throw new Error()` |

### Example
```javascript
function divide(a, b) {
  try {
    if (b === 0) {
      throw new Error("Division by zero!");
    }
    return a / b;
  } catch (error) {
    console.error("Error:", error.message);
    return null;
  } finally {
    console.log("Division operation attempted");
  }
}

console.log(divide(10, 2)); // 5
console.log(divide(10, 0)); // Error: Division by zero!
```

## Special Literals

Keywords representing special primitive values.

| Keyword     | Description                                  | Example |
|-------------|----------------------------------------------|---------|
| `null`      | Represents intentional absence of value      | `let x = null;` |
| `undefined` | Represents uninitialized or missing value    | `let y;` |

### Example
```javascript
// null - intentional absence of value
let user = null; // No user logged in

// undefined - variable declared but not assigned
let data;
console.log(data); // undefined

// undefined vs null
console.log(typeof null);      // "object" (JavaScript quirk)
console.log(typeof undefined); // "undefined"
console.log(null === undefined);  // false
console.log(null == undefined);   // true
```

## Modules

Keywords for working with ES6 modules.

| Keyword   | Description                                  | Example |
|-----------|----------------------------------------------|---------|
| `import`  | Imports functions, objects, or values        | `import { func } from './module'` |
| `export`  | Exports functions, objects, or values        | `export function func() { ... }` |
| `default` | Defines default export/import                | `export default MyClass` |

### Example
```javascript
// Exporting (module.js)
export const PI = 3.14159;
export function add(a, b) {
  return a + b;
}
export default class Calculator {
  multiply(a, b) {
    return a * b;
  }
}

// Importing (main.js)
import Calculator, { PI, add } from './module.js';
import * as MathUtils from './module.js';

console.log(PI);           // 3.14159
console.log(add(2, 3));    // 5
const calc = new Calculator();
console.log(calc.multiply(4, 5)); // 20
```

## Asynchronous Programming

Keywords for handling asynchronous operations.

| Keyword | Description                                    | Example |
|---------|------------------------------------------------|---------|
| `async` | Declares an asynchronous function              | `async function() { ... }` |
| `await` | Pauses execution until promise resolves        | `await promise` |

### Example
```javascript
// Async function
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error("Error fetching data:", error);
  }
}

// Using async/await
async function processData() {
  const data = await fetchData();
  console.log(data);
}

// Promise-based approach
function getData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve("Data received");
    }, 1000);
  });
}

async function main() {
  const result = await getData();
  console.log(result); // "Data received" after 1 second
}
```

## Others

Additional reserved keywords with specific purposes.

| Keyword    | Description                                    | Example |
|------------|------------------------------------------------|---------|
| `yield`    | Pauses and resumes generator function          | `yield value` |
| `debugger` | Stops execution for debugging                  | `debugger;` |
| `with`     | Extends scope chain (deprecated, avoid)        | `with(obj) { ... }` |

### Example
```javascript
// yield - Generator function
function* numberGenerator() {
  yield 1;
  yield 2;
  yield 3;
}

const gen = numberGenerator();
console.log(gen.next().value); // 1
console.log(gen.next().value); // 2
console.log(gen.next().value); // 3

// debugger - Debugging
function calculateTotal(items) {
  let total = 0;
  debugger; // Execution will pause here when dev tools are open
  for (let item of items) {
    total += item.price;
  }
  return total;
}

// with - DO NOT USE (deprecated)
// It makes code harder to understand and optimize
const obj = { a: 1, b: 2 };
with (obj) {
  console.log(a + b); // 3 (not recommended)
}
```

## Future Reserved Words

Keywords reserved for potential future use in JavaScript.

| Keyword       | Description                                |
|---------------|--------------------------------------------|
| `enum`        | Enumeration type (planned feature)         |
| `implements`  | Interface implementation (future use)      |
| `interface`   | Interface definition (future use)          |
| `package`     | Package definition (future use)            |
| `private`     | Private access modifier                    |
| `protected`   | Protected access modifier                  |
| `public`      | Public access modifier                     |
| `static`      | Static member declaration                  |

### Notes
- These keywords are reserved but not yet fully implemented in standard JavaScript
- Some (like `static`) are already used in classes
- Avoid using these as identifiers to prevent future conflicts

### Example
```javascript
// static is already used in classes
class MathUtils {
  static add(a, b) {
    return a + b;
  }
}

console.log(MathUtils.add(5, 3)); // 8 (no instance needed)

// private fields (using # syntax, not the 'private' keyword)
class User {
  #password; // Private field
  
  constructor(password) {
    this.#password = password;
  }
  
  verifyPassword(input) {
    return this.#password === input;
  }
}
```

## Strict Mode Reserved Words

Additional words reserved in strict mode (`'use strict';`).

In strict mode, the following are also reserved:
- `implements`
- `interface`
- `let`
- `package`
- `private`
- `protected`
- `public`
- `static`
- `yield`

### Example
```javascript
'use strict';

// These will cause errors in strict mode:
// var implements = 10;  // SyntaxError
// var interface = 20;   // SyntaxError
// var package = 30;     // SyntaxError

// Strict mode also prevents certain actions
function strictExample() {
  'use strict';
  
  // Error: variable must be declared
  // x = 10; // ReferenceError
  
  // Error: deleting variables not allowed
  // var y = 20;
  // delete y; // SyntaxError
  
  // Error: duplicating parameter names
  // function sum(a, a) { } // SyntaxError
}
```

## Best Practices

### ✅ Do's
1. **Use descriptive names** instead of reserved words
2. **Choose clear alternatives** when a reserved word seems appropriate
3. **Check for conflicts** before naming variables

### ❌ Don'ts
1. **Never use reserved words** as identifiers
2. **Avoid similar names** to reserved words (e.g., `Class`, `Function`)
3. **Don't rely on case sensitivity** to use reserved words

### Example
```javascript
// ❌ Bad (will cause errors)
// let class = "Math";
// let return = 10;
// function if() {}

// ✅ Good (proper alternatives)
let className = "Math";
let returnValue = 10;
function checkIf() {}

// ✅ Better naming
let courseClass = "Math 101";
let apiResponse = { status: 200 };
function validateInput() {}
```

## Common Errors

### Error 1: Using reserved word as variable name
```javascript
// ❌ Error
let for = 10; // SyntaxError: Unexpected token 'for'

// ✅ Solution
let forLoop = 10;
let iterations = 10;
```

### Error 2: Using reserved word as function name
```javascript
// ❌ Error
function switch() { } // SyntaxError: Unexpected token 'switch'

// ✅ Solution
function toggleSwitch() { }
function changeMode() { }
```

### Error 3: Using reserved word as object property (without quotes)
```javascript
// ❌ Error (in strict mode or certain contexts)
const obj = {
  class: "MyClass" // May cause issues
};

// ✅ Solution 1: Use quotes
const obj = {
  "class": "MyClass"
};

// ✅ Solution 2: Better naming
const obj = {
  className: "MyClass"
};
```

## Summary

- **Reserved words** are keywords with special meaning in JavaScript
- They **cannot be used** as identifiers (variable names, function names, etc.)
- All reserved words are **lowercase**
- **Strict mode** adds additional reserved words
- Always use **descriptive alternatives** instead of reserved words
- Understanding reserved words helps **prevent syntax errors** and write cleaner code
