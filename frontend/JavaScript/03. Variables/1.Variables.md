## What is a Variable?

A **variable** is a named memory location used to store data that can be referenced and manipulated in a program.

**Key Characteristics:**
- Stores values temporarily in RAM (stack area)
- Values can be changed during program execution
- Data types can be changed dynamically in JavaScript
- Serves as a reference name for a memory block
- Used to store/hold values for reuse purposes

**Steps for Working with Variables:**
1. **Declare** (create) the variable
2. **Initialize** (set value) into the variable
3. **Use** (get value) from the variable

## Variable Declaration Keywords

In JavaScript, there are **three main ways** to declare variables: `var`, `let`, and `const`.

### 1. var

**Characteristics:**
- **Scope**: Function-scoped (or globally scoped if declared outside a function)
- **Hoisting**: Variables are hoisted to the top of their scope and initialized with `undefined`
- **Re-declaration**: Can be re-declared without errors
- **Re-assignment**: Values can be reassigned
- **Since**: JavaScript 1 (ES1)

#### Syntax:
```javascript
var variableName;              // Declaration
var variableName = value;      // Initialization
```

#### Examples:

**Example 1: Basic Declaration**
```javascript
var a = 10;
console.log(typeof a);  // "number"
console.log(a);         // 10
```

**Example 2: Case Sensitivity**
```javascript
var a = 10;
var A = "abc";
console.log(typeof A);  // "string"
console.log(a, A);      // 10 "abc"
```

**Example 3: Block Scope (var ignores blocks)**
```javascript
{
    var a = 10;
    console.log(a);         // 10
    console.log(typeof a);  // "number"
}
console.log(a);  // 10 (still accessible outside block)
```

**Example 4: Variable Overwriting**
```javascript
var a = 10;
{
    var a = 100;  // Overwrites the outer 'a'
    console.log(a);  // 100
}
console.log(a);  // 100
```

**Example 5: Accessing Outer Variable**
```javascript
var b = 10;
{
    var a = 100;
    console.log(a);  // 100
    console.log(b);  // 10
}
```

**Example 6: Undefined Variables**
```javascript
var b;
console.log(b);  // undefined
```

**Example 7: Multiple Re-declarations**
```javascript
var a = 10;
var a = 20;
var a = 60;
var a = "JavaScript";
console.log(a);  // "JavaScript" (last value)
```

---

### 2. let

**Characteristics:**
- **Scope**: Block-scoped (only available inside the block where it's defined)
- **Hoisting**: Variables are hoisted but NOT initialized (Temporal Dead Zone)
- **Re-declaration**: Cannot be re-declared in the same scope
- **Re-assignment**: Values can be reassigned
- **Since**: ES6 (2015)

#### Syntax:
```javascript
let variableName;              // Declaration
let variableName = value;      // Initialization
```

#### Examples:

**Example 1: Block Scope**
```javascript
{
    let a = "JavaScript";
    console.log(a);  // "JavaScript"
}
// console.log(a);  // ReferenceError: a is not defined
```

**Example 2: Block-Scoped Variables**
```javascript
let a = 100;
{
    let a = 20;  // Different variable (block-scoped)
    console.log(a);  // 20
}
console.log(a);  // 100
```

**Example 3: Combining var and let**
```javascript
var a = 100;
{
    let a = 20;  // Block-scoped, doesn't affect outer 'var a'
    console.log(a);  // 20
}
console.log(a);  // 100
```

**Example 4: Cannot Re-declare with var**
```javascript
{
    let a = 100;
    // var a = 20;  // SyntaxError: Identifier 'a' has already been declared
}
```

**Example 5: Proper Block Scoping**
```javascript
{
    let a;
    a = 10;
    console.log(a);  // 10
}
```

**Example 6: Re-assignment is Allowed**
```javascript
let d = 5;
d = 15;  // Allowed
console.log(d);  // 15
```

**Example 7: Temporal Dead Zone**
```javascript
// console.log(d);  // ReferenceError: Cannot access 'd' before initialization
let d = 5;
console.log(d);  // 5
```

---

### 3. const

**Characteristics:**
- **Scope**: Block-scoped, like `let`
- **Hoisting**: Hoisted but NOT initialized (Temporal Dead Zone)
- **Re-declaration**: Cannot be re-declared
- **Re-assignment**: Cannot be reassigned
- **Immutability**: Value cannot be changed, but object properties can be modified
- **Initialization**: **MUST** be initialized at declaration
- **Since**: ES6 (2015)

#### Syntax:
```javascript
const variableName = value;  // Must initialize immediately
```

#### Examples:

**Example 1: Must Initialize**
```javascript
const c = 42;  // Valid
// const d;    // SyntaxError: Missing initializer in const declaration
```

**Example 2: Basic Usage**
```javascript
{
    const a = "John Doe";
    console.log(a);  // "John Doe"
}
```

**Example 3: Cannot Reassign**
```javascript
{
    const a = "John Doe";
    // a = "AB";  // TypeError: Assignment to constant variable
}
```

**Example 4: Cannot Reassign Numbers**
```javascript
{
    const a = 100;
    // a = 2500;  // TypeError: Assignment to constant variable
}
```

**Example 5: Initialization Error**
```javascript
{
    // const a;  // SyntaxError: Missing initializer in const declaration
    
    const b = 10;  // Correctly initialized
    console.log(typeof b);  // "number"
    console.log(b);         // 10
}
```

**Example 6: const with Objects (Mutable Properties)**
```javascript
const person = { name: "John", age: 30 };
person.age = 31;  // Allowed (modifying properties)
console.log(person.age);  // 31

// person = {};  // TypeError: Assignment to constant variable
```

**Example 7: const with Arrays (Mutable Elements)**
```javascript
const numbers = [1, 2, 3];
numbers.push(4);  // Allowed (modifying array contents)
console.log(numbers);  // [1, 2, 3, 4]

// numbers = [];  // TypeError: Assignment to constant variable
```

## Key Differences Between var, let, and const

| Feature | var | let | const |
|---------|-----|-----|-------|
| **Scope** | Function-scoped | Block-scoped | Block-scoped |
| **Hoisting** | Hoisted & initialized with `undefined` | Hoisted but NOT initialized (TDZ) | Hoisted but NOT initialized (TDZ) |
| **Re-declaration** | ‚úÖ Allowed | ‚ùå Not allowed | ‚ùå Not allowed |
| **Re-assignment** | ‚úÖ Allowed | ‚úÖ Allowed | ‚ùå Not allowed |
| **Initialization** | Optional | Optional | **Required** |
| **Introduced** | ES1 | ES6 (2015) | ES6 (2015) |
| **Usage** | Avoid in modern JS | Preferred for changeable values | Preferred for constants |

## Variable Naming Rules

### Valid Naming Rules:

1. ‚úÖ Must start with:
   - Letter (a-z, A-Z)
   - Underscore (_)
   - Dollar sign ($)

2. ‚úÖ After first character, can contain:
   - Letters
   - Digits (0-9)
   - Underscore (_)
   - Dollar sign ($)

3. ‚úÖ Case-sensitive (`myVar` ‚â† `myvar`)

4. ‚úÖ No spaces allowed (use camelCase or snake_case)

### Examples:

#### Valid Variable Names:
```javascript
var eid;
var total;
var _b;
var book_id;
var a1;
var $price;
var studentId;
var userName;
```

#### Invalid Variable Names:
```javascript
// var 1a;        // Cannot start with digit
// var a@;        // @ not allowed
// var #b;        // # not allowed
// var book id;   // Space not allowed
// var case;      // Reserved keyword
```

### Reserved Keywords to Avoid:
Cannot use JavaScript reserved words as variable names:
```plaintext
break, case, catch, class, const, continue, debugger, default, delete,
do, else, export, extends, finally, for, function, if, import, in,
instanceof, let, new, return, static, super, switch, this, throw, try,
typeof, var, void, while, with, yield
```

## Variable Scope

### Global Scope

Variables declared outside any function or block are **global variables**.

```javascript
var globalVar = "I'm global";

function test() {
    console.log(globalVar);  // Accessible
}

test();
console.log(globalVar);  // Accessible
```

**Global with `window` object:**
```javascript
var globalVar = 10;
console.log(window.globalVar);  // 10
```

### Function Scope

Variables declared with `var` inside a function are **function-scoped**.

```javascript
function calculate() {
    var result = 100;  // Function-scoped
    console.log(result);  // 100
}

calculate();
// console.log(result);  // ReferenceError: result is not defined
```

### Block Scope

Variables declared with `let` or `const` inside blocks `{}` are **block-scoped**.

```javascript
if (true) {
    let blockVar = "I'm block-scoped";
    const BLOCK_CONST = 42;
    console.log(blockVar);      // Accessible
    console.log(BLOCK_CONST);   // Accessible
}

// console.log(blockVar);      // ReferenceError
// console.log(BLOCK_CONST);   // ReferenceError
```

## Hoisting

**Hoisting** is JavaScript's behavior of moving declarations to the top of their scope before code execution.

### var Hoisting:
```javascript
console.log(x);  // undefined (hoisted, initialized with undefined)
var x = 5;
console.log(x);  // 5

// Interpreted as:
// var x;
// console.log(x);
// x = 5;
```

### let/const Hoisting (Temporal Dead Zone):
```javascript
// console.log(y);  // ReferenceError: Cannot access 'y' before initialization
let y = 10;
console.log(y);  // 10
```

**Temporal Dead Zone (TDZ):** The period between entering scope and variable declaration where the variable cannot be accessed.

## Implicit Variable Declaration

Assigning a value to an undeclared variable creates an **implicit global variable** (in non-strict mode).

### Example:
```javascript
function test() {
    implicitVar = 100;  // Creates global variable (BAD PRACTICE!)
}

test();
console.log(implicitVar);  // 100
```

### Strict Mode Prevents This:
```javascript
'use strict';

function test() {
    // implicitVar = 100;  // ReferenceError: implicitVar is not defined
}
```

**Why Avoid Implicit Declarations:**
- Creates global variables unintentionally
- Causes naming conflicts
- Makes code harder to maintain
- Not block-scoped

**Always use `let`, `const`, or `var`:**
```javascript
function test() {
    let explicitVar = 100;  // Properly declared
}
```

## Loosely Typed vs Dynamically Typed

### Loosely Typed

JavaScript doesn't require explicit data type declarations. The data type is determined automatically.

```javascript
var x = 10;        // Number
x = "Hello";       // Now String
x = true;          // Now Boolean
x = { id: 1 };     // Now Object
```

### Dynamically Typed

Variables can be used without prior declaration (not recommended).

```javascript
// Without declaration (implicit global)
result = 10 + 20;
console.log(result);  // 30
```

**Better Practice:**
```javascript
let result = 10 + 20;  // Explicitly declared
console.log(result);  // 30
```

## Examples and Use Cases

### Example 1: Combining Variables
```javascript
var a = 10;
var b = 20;
var c = 30;

let d = a + b + c;
console.log(d);  // 60
```

### Example 2: Block Scope Calculation
```javascript
var a = 10;
{
    let b = 20;
    const c = 30;
    let d = a + b + c;
    console.log(d);  // 60
}
```

### Example 3: Nested Blocks
```javascript
var a = 1;
{
    let b = 2;
    const c = 3;
    {
        a = 2;
        let b = 4;  // Different 'b' (block-scoped)
        const c = 5;  // Different 'c' (block-scoped)
        let x = a + b + c;
        console.log(x);  // 11
    }
    b = 10;
    let y = a + b + c;
    console.log(y);  // 15
}
```

### Example 4: Implicit Global Variables
```javascript
a = 1;
{
    b = 10;
    console.log(a, b);  // 1, 10 (both global)
}
console.log(a, b);  // 1, 10 (accessible outside)
```

## Best Practices

### ‚úÖ Do:

1. **Use `const` by default**
   ```javascript
   const PI = 3.14159;
   const API_URL = "https://api.example.com";
   ```

2. **Use `let` when value needs to change**
   ```javascript
   let counter = 0;
   counter++;
   ```

3. **Avoid `var` in modern JavaScript**
   - Use `let` or `const` instead for better scoping

4. **Always declare variables**
   ```javascript
   let userName = "John";  // Good
   // userName = "John";  // Bad (implicit global)
   ```

5. **Use descriptive names**
   ```javascript
   const userAge = 25;  // Good
   // const a = 25;     // Bad
   ```

6. **Initialize const at declaration**
   ```javascript
   const MAX_SIZE = 100;  // Required
   ```

7. **Use camelCase for variable names**
   ```javascript
   let firstName = "John";
   let totalAmount = 1000;
   ```

8. **Use UPPERCASE for constants**
   ```javascript
   const MAX_USERS = 100;
   const API_KEY = "abc123";
   ```

9. **Enable strict mode**
   ```javascript
   'use strict';
   // Rest of your code
   ```

10. **Declare variables at the top**
    ```javascript
    function calculate() {
        let x, y, result;
        x = 10;
        y = 20;
        result = x + y;
    }
    ```

## Common Mistakes to Avoid

### ‚ùå Don't:

1. **Don't use var in modern JavaScript**
   ```javascript
   // var x = 10;  // Avoid
   let x = 10;     // Prefer
   ```

2. **Don't forget to declare variables**
   ```javascript
   // BAD:
   count = 5;  // Creates implicit global
   
   // GOOD:
   let count = 5;
   ```

3. **Don't reassign const variables**
   ```javascript
   const MAX = 100;
   // MAX = 200;  // TypeError
   ```

4. **Don't leave const uninitialized**
   ```javascript
   // const value;  // SyntaxError
   const value = 10;  // Correct
   ```

5. **Don't re-declare let/const in same scope**
   ```javascript
   let x = 10;
   // let x = 20;  // SyntaxError
   x = 20;  // Correct (reassignment)
   ```

6. **Don't access variables before declaration**
   ```javascript
   // console.log(x);  // ReferenceError
   let x = 5;
   ```

7. **Don't use reserved keywords**
   ```javascript
   // let class = "Math";  // SyntaxError
   let className = "Math";  // Correct
   ```

8. **Don't mix var with let/const in the same scope**
   ```javascript
   // Confusing:
   var a = 1;
   let b = 2;
   
   // Better: Choose one style
   let a = 1;
   let b = 2;
   ```

## Summary

### Quick Decision Guide:

```
Need a constant value? ‚Üí Use const
Need to reassign value? ‚Üí Use let
Working with legacy code? ‚Üí Understand var (but prefer let/const)
```

### Modern JavaScript Best Practice:
```javascript
// Default to const
const API_URL = "https://api.example.com";
const MAX_RETRIES = 3;

// Use let when value needs to change
let counter = 0;
let userName = "";

// Avoid var (unless maintaining legacy code)
// var oldStyle = "avoid in new code";
```

### Scope Summary:
- **var**: Function-scoped (leaks out of blocks)
- **let**: Block-scoped (stays in blocks)
- **const**: Block-scoped (stays in blocks, can't reassign)

---

**Remember**: In modern JavaScript development, **prefer `const` by default**, use `let` when you need to reassign, and **avoid `var`** due to its unpredictable scoping behavior.
