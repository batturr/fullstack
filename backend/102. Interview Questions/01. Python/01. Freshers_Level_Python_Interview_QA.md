## Python Basics & Syntax

### Q1. What is Python? What are its key features?

**Answer:**

Python is a high-level, interpreted, general-purpose programming language created by Guido van Rossum and first released in 1991. It emphasizes readability through significant whitespace (indentation) and a clear, expressive syntax that often reads almost like English. Because it is high-level, you work with abstractions like lists and dictionaries without managing memory manually, which speeds up development for beginners and professionals alike.

Python runs on many platforms (Windows, macOS, Linux) and is widely used for web development, data science, automation, scripting, and education. The language has a large standard library (тАЬbatteries includedтАЭ) and an enormous ecosystem of third-party packages installable via pip. Python 3 is the current line; Python 2 reached end-of-life in 2020.

**Key Features:**
- Readable syntax with indentation-based blocks
- Dynamically typed: variable types are inferred at runtime
- Interpreted execution via the CPython reference implementation (and others like PyPy)
- Multi-paradigm: procedural, object-oriented, and functional styles
- Extensive standard library and strong community support

```python
# Simple illustration: readable, expressive code
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet("Python"))  # Hello, Python!
```

### Q2. What is the difference between Python 2 and Python 3?

**Answer:**

Python 3 is the actively maintained language line; Python 2 is legacy and should not be used for new projects. The most visible difference is **print**: in Python 2, `print` was a statement (`print "hi"`), while in Python 3 it is a function (`print("hi")`). **Integer division** changed: in Python 2, `/` between integers truncated toward zero; in Python 3, `/` always performs true division and `//` is floor division.

**Unicode** handling is first-class in Python 3 (`str` is Unicode); Python 2 had separate `str` and `unicode` types, which caused many encoding bugs. **Dictionary methods** like `.keys()`, `.values()`, and `.items()` return view objects in Python 3, not lists. Syntax additions such as `async`/`await`, f-strings, the `match` statement, and structural pattern matching exist only in Python 3.

For interviews, state clearly: use Python 3.12+ for new code, migrate old Python 2 code with tools like `2to3` if still needed.

```python
# Python 3 division
print(5 / 2)   # 2.5
print(5 // 2)  # 2

# Python 3 strings are Unicode by default
s = "caf├й"
print(len(s))  # character count, not byte count (for str)
```

### Q5. How is Python an interpreted language?

**Answer:**

**Interpreted** usually means source code is executed without a separate manual compilation step to machine code that you run later. In practice, **CPython** (the standard implementation) compiles `.py` source to **bytecode** (`.pyc` in `__pycache__`), then the **CPython virtual machine** interprets that bytecode. So Python is often described as тАЬinterpretedтАЭ from the developerтАЩs perspective, but under the hood there is a compile-to-bytecode phase.

This contrasts with languages like C, where you compile to a native executable ahead of time. Other implementations exist: **PyPy** uses a JIT; **Jython** targets the JVM. The key interview point: you typically run `python script.py` and the interpreter handles compilation and execution transparently; bytecode is cached for faster startup on subsequent runs.

```python
# You run source directly; CPython handles bytecode internally
import dis

def add(a, b):
    return a + b

dis.dis(add)  # shows bytecode instructions (educational)
```

### Q6. What is the difference between dynamically typed and statically typed languages?

**Answer:**

In **statically typed** languages (e.g., Java, C++), variable and expression types are mostly checked **at compile time**. You often declare types explicitly, and assigning incompatible types is a compile error. In **dynamically typed** languages like Python, types are associated with **values** (objects), not variable names, and checks occur **at runtime** as operations execute.

Python variables are names bound to objects; the same name can refer to an `int` and later a `str`. This speeds prototyping but means type errors may only appear when a branch runs. Since **PEP 484**, you can add **type hints** (`def f(x: int) -> str`) and use **mypy** or Pyright for static analysis without changing runtime behaviorтАФbridging both worlds.

```python
# Dynamic: x can point to different types over time
x = 10
x = "hello"  # valid in Python

# Optional static checking with hints (not enforced at runtime by default)
def double(n: int) -> int:
    return n * 2
```

### Q7. What is indentation in Python and why is it important?

**Answer:**

Unlike languages that use `{}` for blocks, Python uses **leading whitespace** (spaces or tabs, but **spaces are strongly preferred**) to define block structure. After a line ending with `:`, the next lines must be indented consistently to belong to that block. Increasing indentation starts a nested block; decreasing indentation ends the current block.

Indentation is important because it is **syntactic**: inconsistent indentation causes `IndentationError`. It enforces a uniform visual structure and is a major reason Python code often looks clean. PEP 8 recommends **4 spaces** per level; mixing tabs and spaces is disallowed in Python 3. Editors can be configured to insert spaces when you press Tab.

```python
def example(x):
    if x > 0:
        print("positive")
        if x > 10:
            print("large")
    else:
        print("non-positive")
```

### Q8. What are comments in Python? How do you write single-line and multi-line comments?

**Answer:**

**Comments** are text ignored by the interpreter; they document intent, explain non-obvious logic, or temporarily disable code. A **single-line comment** starts with `#` and runs to the end of the line. There is no separate multi-line comment syntax like `/* */`; instead, use a **multi-line string** (often triple-quoted) as a statement by itself, or multiple `#` lines.

**Docstrings** are string literals as the first statement in a module, class, or function; tools use them for documentation (`help()`, Sphinx). Do not confuse a triple-quoted string used only for commenting inside expressions with a docstringтАФplacement matters.

```python

# lines with
# leading #

(not a docstring unless it's the first line of a definition).
"""

    """This docstring describes add()."""
    return a + b
```

### Q9. What is a variable in Python? How do you declare one?

**Answer:**

In Python, a **variable** is a **name** (identifier) bound to an object in a namespace. You do not тАЬdeclareтАЭ a type or allocate memory explicitly; you **assign** with `=` to create or rebind a name. Names must follow rules: letters, digits, underscores; cannot start with a digit; reserved keywords (`if`, `class`, etc.) are forbidden.

Multiple names can reference the same object (`a = b`). Assignment never copies objects automatically for mutable typesтАФboth names may point to the same list. тАЬDeclarationтАЭ in the C sense does not exist; the first assignment creates the binding (in local or global scope depending on context).

```python
count = 0           # name 'count' bound to int 0
message = "hello"   # rebound to a str object
x = y = []          # careful: same list object shared
x.append(1)
print(y)            # [1]
```

### Q10. What is type casting in Python? Give examples.

**Answer:**

**Type casting** means converting a value from one type to another. Python uses **constructors** rather than special cast syntax: `int()`, `float()`, `str()`, `bool()`, `list()`, `tuple()`, `set()`, etc. These calls may fail if the value cannot be meaningfully converted (e.g., `int("abc")` raises `ValueError`).

Some conversions are **lossy** (`float` to `int` truncates toward zero). **Rounding** for floats should use `round()` if you need nearest integer behavior before casting. For collections, `list("abc")` gives `['a','b','c']`. Explicit conversion is preferred over relying on implicit behavior (Python 3 removed much implicit mixing, e.g., `str` + `int` is an error).

```python
print(int("42"))        # 42
print(int(3.9))         # 3 (truncates toward zero)
print(float("3.14"))    # 3.14
print(str(100))         # "100"
print(bool(0))          # False; bool("") -> False
print(list("hi"))       # ['h', 'i']
```

## Data Types & Variables

### Q11. What is the difference between mutable and immutable data types?

**Answer:**

**Immutable** objects cannot be changed in place after creation; any operation that тАЬmodifiesтАЭ them returns a **new** object. Examples: `int`, `float`, `str`, `tuple`, `frozenset`, `bytes`. **Mutable** objects can be altered without replacing the object identity: `list`, `dict`, `set`, and most user-defined classes by default.

This matters for **default arguments** (`def f(x=[]):` shares one list), **hashing** (mutable types are not hashable and cannot be dict keys), and **thread safety** (mutable shared state needs locks). Use `id()` and `is` to verify whether you mutated in place or created a new object.

```python
s = "ab"
s += "c"          # new str object; old "ab" unchanged if unreferenced
lst = [1, 2]
lst.append(3)     # same list object mutated
print(id(lst))      # unchanged after append
```

### Q12. What are numbers in Python? Explain int, float, and complex.

**Answer:**

PythonтАЩs numeric tower includes **`int`**: arbitrary-precision integers (no fixed overflow like 32-bit in pure Python math). **`float`** maps to C double (typically IEEE 754 binary64)тАФfinite precision, so `0.1 + 0.2` may not equal `0.3` exactly; use `decimal.Decimal` for exact decimal arithmetic when needed. **`complex`** has real and imaginary parts as floats (`3 + 4j`).

Literals: `42` int, `3.14` float, `2e-3` float in scientific notation, `1+2j` complex. Operations promote types (`int` + `float` тЖТ `float`). For interviews, mention **integer division** `//` vs true division `/`, and that `int` has no size limit beyond memory.

```python
a = 10**100        # huge int, no overflow
b = 0.1 + 0.2
print(b)           # 0.30000000000000004
c = complex(1, 2)
print(c.real, c.imag)
```

### Q13. What is None in Python?

**Answer:**

`None` is a singleton of type `NoneType` representing the **absence of a value** or a **null** reference. Functions with no explicit `return` return `None`. It is a valid dictionary value and a common default for optional parameters (`def f(x=None)`). Use **`is None`** to test for `None`, not `== None`, because `is` checks identity and is idiomatic and slightly faster.

Do not confuse `None` with `False`, `0`, or `[]`тАФthey are different objects; `bool(None)` is `False` in a boolean context, but `if x is None` is precise when distinguishing тАЬnot providedтАЭ from тАЬempty list.тАЭ

```python

    if x is None:
        return None
    return x + 1

print(maybe_increment(None))  # None
```

### Q14. What is the difference between `is` and `==`?

**Answer:**

**`==`** checks **value equality** by calling `__eq__`тАФwhether two objects represent the same data according to their typeтАЩs rules. **`is`** checks **object identity**: whether two references point to the **same object** in memory (`id(a) == id(b)`).

For small integers and interned strings, `is` may appear to work for equality, but that is an implementation detailтАФalways use `==` for value comparison of numbers and strings. Use **`is`** for singletons: `None`, `True`, `False`, and sometimes `is` with `Enum` members. Mutable objects with equal contents may have `==` True but `is` False.

```python
a = [1, 2]
b = [1, 2]
print(a == b)  # True (same contents)
print(a is b)  # False (different list objects)

x = None
print(x is None)  # True (idiomatic)
```

### Q15. What is the `type()` function?

**Answer:**

`type(obj)` returns the **class** of an objectтАФusually the type you expect, e.g., `type(3)` is `<class 'int'>`. For **new-style classes** (all in Python 3), `type()` is reliable for built-ins and user classes. For metaprogramming, `type(name, bases, dict)` also **creates** a new class dynamically.

In interviews, contrast `type(x) is list` with `isinstance(x, list)`тАФthe latter handles **subclassing** correctly. `type()` does not consider inheritance for equality checks the way `isinstance()` does.

```python
print(type(42))           # <class 'int'>
print(type([]))           # <class 'list'>

class Dog:
    pass

d = Dog()
print(type(d))            # <class '__main__.Dog'>
```

### Q16. What is the `id()` function in Python?

**Answer:**

`id(obj)` returns an integer that is **unique among simultaneously existing objects** (in CPython, it is often the objectтАЩs memory address). If two variables have the same `id`, they reference the **same object**. After an object is garbage-collected, its `id` may be reusedтАФdo not store `id` long-term as a universal unique key across time.

Use `id()` to debug aliasing and to understand in-place mutations versus rebinding. It complements `is`, which compares identity directly.

```python
a = [1]
b = a
print(id(a) == id(b))  # True
b = b + [2]            # creates new list; a unchanged
print(a, b)
```

### Q17. What is string interning in Python?

**Answer:**

**String interning** means the interpreter may store **only one copy** of certain string literals that look like identifiers, so multiple references share one objectтАФsaving memory and making `is` accidentally true for those strings. CPython interns some strings created at compile time that resemble identifiers; behavior is **implementation-dependent** and should not be relied upon for program logic.

**Never** use `is` to compare arbitrary strings for equalityтАФalways use `==`. Interning is an optimization detail; interviews may mention it when discussing `id()` and small string behavior.

```python
a = "hello"
b = "hello"
print(a == b)   # always use this
# a is b might be True for some literals in CPythonтАФdon't depend on it
```

### Q18. What is the difference between `type()` and `isinstance()`?

**Answer:**

`type(obj) is SomeClass` is True only for **exact** type matchesтАФit returns `False` for instances of **subclasses**. `isinstance(obj, SomeClass)` returns `True` if `obj` is an instance of `SomeClass` **or any subclass**. For built-in abstract bases (e.g., `numbers.Integral`), `isinstance` works with registered virtual subclasses.

Best practice: prefer **`isinstance()`** for type checks in polymorphic code; use `type()` when you truly need the concrete class or for metaclass tricks.

```python
class Animal: ...
class Dog(Animal): ...

d = Dog()
print(isinstance(d, Animal))  # True
print(type(d) is Animal)      # False
```

### Q19. How does Python handle memory management?

**Answer:**

Python uses **reference counting**: every object stores a count of references; when it drops to zero, the object is deallocated immediately. This handles most cleanup. A **cycle detector** (generational garbage collection) breaks reference **cycles** (e.g., two objects referencing each other) that reference counting alone cannot free.

The **`gc` module** exposes the garbage collector for advanced use. **CPython**тАЩs allocator also pools small objects for efficiency. Developers rarely manually free memory; deleting references (`del x` removes a name binding, not necessarily immediate object destruction unless refcount hits zero).

```python
import gc
gc.collect()  # usually unnecessary; forces collection of cycles
```

### Q20. What is garbage collection in Python?

**Answer:**

**Garbage collection** reclaims memory occupied by objects no longer reachable from the program. In CPython, **reference counting** is the primary mechanism; the **cyclic GC** supplements it for cycles involving container objects. Generations reduce pause times by focusing on young objects first.

You typically do not call the GC explicitly. Avoid creating uncollectable cycles with `__del__` methods in cyclesтАФthose can complicate cleanup. For large long-running apps, understanding GC helps when profiling memory leaks (objects kept alive unintentionally).

```python
# Cycle example (eventually collected by cyclic GC)
a = []
b = []
a.append(b)
b.append(a)
del a, b
```

## Operators

### Q21. What are the different types of operators in Python?

**Answer:**

Python supports several operator categories: **Arithmetic** (`+`, `-`, `*`, `/`, `//`, `%`, `**`), **Comparison** (`==`, `!=`, `<`, `>`, `<=`, `>=`), **Assignment** (`=`, `+=`, `-=`, etc.), **Logical** (`and`, `or`, `not`), **Bitwise** (`&`, `|`, `^`, `~`, `<<`, `>>`), **Identity** (`is`, `is not`), **Membership** (`in`, `not in`), and **Walrus** (`:=`) for assignment expressions (3.8+).

Unary operators include `-x`, `+x`, `~x`. **Matrix multiplication** `@` exists for numeric types supporting `__matmul__`. Understanding precedence (see Q27) prevents subtle bugs.

```python
print(2 ** 10)      # exponentiation
print(7 // 3)       # floor division
print(7 % 3)        # remainder
```

### Q22. What is the difference between `/` and `//` in Python?

**Answer:**

In Python 3, **`/`** is **true division**: dividing two integers still yields a **float** (`5 / 2` тЖТ `2.5`). **`//`** is **floor division**: it floors the result toward negative infinity and returns an `int` if both operands are ints (otherwise float floor behavior).

For negative numbers, floor division differs from truncation: `-5 // 2` is `-3`, while `int(-5 / 2)` might truncate toward zero in a C castтАФPythonтАЩs `//` consistently floors.

```python
print(5 / 2)    # 2.5
print(5 // 2)   # 2
print(-5 // 2)  # -3
```

### Q23. What is the `%` (modulo) operator?

**Answer:**

The `%` operator returns the **remainder** of division. For integers, `a % b` has the same sign as **b** in Python (unlike some languages where remainder matches the dividend). It is widely used for **cyclic indexing**, checking **even/odd** (`n % 2`), and **wrapping** values.

For floats, `%` uses the same definition relative to float division. For negative operands, behavior follows the mathematical relation `(a // b) * b + a % b == a`.

```python
print(7 % 3)     # 1
print(-7 % 3)    # 2  (Python convention)
print(10 % 2)    # 0 -> even
```

### Q24. What are logical operators in Python?

**Answer:**

Python has **`and`**, **`or`**, and **`not`**. They operate on truthiness: values are converted with `bool()` in a boolean context, but **`and`/`or` return one of the operands**, not necessarily `True`/`False`тАФshort-circuit evaluation stops as soon as the result is determined (`0 and expensive()` never calls `expensive()`).

**`not`** always returns a proper boolean. **Precedence**: `not` binds tightest, then `and`, then `or`. Use parentheses for clarity in complex conditions.

```python
print(3 and 5)       # 5 (last evaluated)
print(0 or "default")  # "default"
print(not [])        # True
```

### Q25. What are bitwise operators in Python?

**Answer:**

Bitwise operators work on integersтАЩ binary representations: **`&`** AND, **`|`** OR, **`^`** XOR, **`~`** NOT (twoтАЩs complement), **`<<`** and **`>>`** shifts. They are used in low-level flags, cryptography, and performance-sensitive bit packing. Python integers have **unlimited precision**, so `~` follows twoтАЩs complement semantics in an infinite sign extension model.

For **non-integers**, these operators call dunder methods like `__and__`. Use with nonnegative integers for clarity unless you understand signed bit patterns.

```python
print(0b1100 & 0b1010)   # 0b1000
print(8 << 1)            # 16
print(bin(~0))           # '-0b1' (infinite ones in two's complement model)
```

### Q26. What is the ternary operator in Python?

**Answer:**

PythonтАЩs conditional expression is **`x if condition else y`**. It evaluates `condition` first, then returns **`x` if true**, else **`y`**. Unlike CтАЩs `?:`, both branchesтАЩ expressions may be evaluated only lazily in the sense that only the chosen branch is evaluated (the other is skipped).

Use parentheses when nesting to avoid readability issues. It is expression-level, so it can appear inside comprehensions or function arguments.

```python
score = 85
grade = "pass" if score >= 50 else "fail"
print(grade)
```

### Q27. What is operator precedence in Python?

**Answer:**

**Operator precedence** determines evaluation order when parentheses are omitted: e.g., `**` binds tighter than unary `-`, which binds tighter than `*`, than `+`, than comparisons, than `not`, `and`, `or`. **Parentheses** override precedence and should be used when uncertain.

Full tables are in the Python language reference. Common pitfall: mixing `and`/`or` with comparisonsтАФ`a or b == c` parses as `a or (b == c)`, not `(a or b) == c`.

```python
print(2 + 3 * 4)       # 14, not 20
print(not False and True)  # True
```

## Control Flow

### Q28. Explain if, elif, and else statements in Python.

**Answer:**

The **`if`** statement runs a block when a condition is truthy. **`elif`** (else if) chains additional tests; only the **first** matching branch runs. **`else`** runs if no prior condition matched. Conditions are expressions evaluated in order; there is no `switch` until **`match`** (3.10+).

Python uses **`:`** and indentation for blocks. Empty blocks require **`pass`**. Truthiness: `None`, `0`, `False`, empty collections, and empty strings are falsy; most other values are truthy.

```python
x = 10
if x < 0:
    print("negative")
elif x == 0:
    print("zero")
else:
    print("positive")
```

### Q29. What are loops in Python? Explain for and while loops.

**Answer:**

Loops repeat code. A **`while`** loop tests a condition **before** each iteration; it can run zero or many times and risks infinite loops if the condition never becomes false. A **`for`** loop iterates over an **iterable** (list, range, file lines, dict keys, etc.)тАФit is the preferred way to traverse known sequences.

`for` hides index management; use `range(len(seq))` only when you need indices, or better `enumerate(seq)`. Both loops support **`break`** (exit loop), **`continue`** (skip to next iteration), and **`else`** clauses (Q34).

```python
for ch in "abc":
    print(ch)

n = 3
while n > 0:
    print(n)
    n -= 1
```

### Q30. What is the difference between `break`, `continue`, and `pass`?

**Answer:**

**`break`** exits the **innermost** active loop (`for`/`while`), skipping any loop `else` clause. **`continue`** skips the rest of the current iteration and jumps to the loopтАЩs next iteration (still in the same loop). **`pass`** is a **no-op** placeholder: it does nothing but satisfies syntax where a statement is required (empty class or stub function).

`break`/`continue` do not apply to `if` unless inside a loop. In `try`/`finally`, `break`/`continue` still run `finally` before leaving.

```python
for i in range(5):
    if i == 2:
        continue
    if i == 4:
        break
    print(i)
```

### Q31. What is the `range()` function?

**Answer:**

`range(stop)` yields integers from `0` to `stop-1`. `range(start, stop)` uses `start` inclusive and `stop` exclusive. `range(start, stop, step)` adds a stride; step can be negative for countdowns. In Python 3, **`range` returns a range object** (lazy, memory-efficient), not a listтАФwrap with `list(range(...))` if you need materialized values.

`range` is ideal for fixed iteration counts and indexing. It supports `len()`, membership tests, and indexing without building a full list.

```python
print(list(range(3)))           # [0, 1, 2]
print(list(range(10, 0, -2)))   # [10, 8, 6, 4, 2]
```

### Q32. How does the `enumerate()` function work?

**Answer:**

`enumerate(iterable, start=0)` yields pairs **`(index, value)`** for each item, so you avoid manual `range(len(x))`. The optional **`start`** argument sets the initial index (useful for 1-based display). It returns an **iterator**; consume once or wrap in `list()`.

Prefer `enumerate` over index counters for clarity and to iterate any iterable, not only sequences.

```python
names = ["a", "b", "c"]
for i, name in enumerate(names, start=1):
    print(i, name)
```

### Q33. What is a nested loop? Give an example.

**Answer:**

A **nested loop** places one loop inside another. The inner loop completes all iterations for **each** iteration of the outer loopтАФcomplexity often multiplies (e.g., O(n┬▓) for n├Чn matrix traversal). Use meaningful names and avoid deep nesting when possible (extract functions or use comprehensions carefully).

Nested loops appear in 2D grids, bubble sort, combinations, and parsing tables.

```python
for i in range(3):
    for j in range(2):
        print(i, j)
# prints (0,0),(0,1),(1,0),(1,1),(2,0),(2,1)
```

---

### Q34. What is the `else` clause in loops?

**Answer:**

Both **`for`** and **`while`** can have an **`else`** clause executed when the loop **completes without `break`**. If `break` exits the loop, the `else` is **skipped**. This pattern distinguishes тАЬfound somethingтАЭ vs тАЬexhausted iterableтАЭ without extra flags.

Example: search a list for a valueтАФ`else` runs if not found. Misunderstanding this feature is common in interviews, so explain it clearly.

```python
for n in [2, 4, 6]:
    if n % 2 != 0:
        break
else:
    print("all even")  # runs: no break
```

---

### Q35. What is the `match` statement in Python 3.10+?

**Answer:**

The **`match`/`case`** statement performs **structural pattern matching**. You `match` a subject value against **`case`** patterns: literals, capture names, wildcards `_`, sequences, mappings, and class patterns. It is more expressive than long `if`/`elif` chains for shaped data (AST nodes, messages, dataclasses).

Guards use `if` after the pattern. Order mattersтАФfirst matching case wins. It is **not** a simple switch on values only; patterns can destructure.

```python
def http_status(code):
    match code:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case _:
            return "Other"

print(http_status(404))
```

## Strings

### Q36. How do you create strings in Python?

**Answer:**

Strings are **immutable** sequences of Unicode code points. Create with **single** `'...'` or **double** `"..."` quotes; style guides often pick one and stick to it. **Triple quotes** `'''` or `"""` span multiple lines and preserve newlines. Adjacent string literals concatenate implicitly: `"a" "b"` тЖТ `"ab"`.

**Raw strings** `r"\n"` treat backslashes literally (useful for regex paths). **f-strings** `f"{x}"` interpolate expressions (3.6+).

```python
s1 = 'hello'
s2 = "world"
s3 = """line1
line2"""
path = r"C:\new\folder"
print(f"{s1} {s2}")
```

---

### Q37. What is string slicing?

**Answer:**

**Slicing** extracts substrings with `s[start:stop:step]`. **`stop` is exclusive**. Omitted `start` defaults to beginning, omitted `stop` to end. Negative indices count from the end (`-1` is last character). A **step** of `-1` reverses; `s[::-1]` is idiomatic reversal.

Slicing always returns a **new** string; out-of-range slices clamp safely (no error for valid slice bounds logic).

```python
s = "Python"
print(s[0:2])    # Py
print(s[-2:])    # on
print(s[::2])    # Pto
```

---

### Q38. What are f-strings in Python?

**Answer:**

**Formatted string literals** (f-strings) prefix with `f` or `F` and embed `{expression}` evaluated at runtime. They are readable and fastтАФpreferred over old `%` formatting and often over `str.format`. Since Python 3.8, `f"{x=}"` prints debug-style `x=value`.

Escape braces with `{{` and `}}`. Format specifiers go after `:` inside braces: `f"{pi:.2f}"`.

```python
name = "Ada"
print(f"Hello, {name}!")
print(f"{2 + 2 = }")
```

---

### Q39. What are common string methods in Python?

**Answer:**

Common methods include: **`strip()`** / **`lstrip()`** / **`rstrip()`** trim whitespace; **`lower()`** / **`upper()`** casefolding; **`startswith()`** / **`endswith()`** prefixes/suffixes; **`split()`** and **`rsplit()`** tokenize; **`join()`** concatenates iterables of strings; **`replace()`** substitutes substrings; **`find()`** / **`index()`** locate substrings (Q41).

**`isdigit()`**, **`isalpha()`**, **`isalnum()`** test character classes. Strings are immutableтАФmethods return new strings.

```python
print("  hi  ".strip())
print(",".join(["a", "b", "c"]))
print("hello world".replace("world", "Python"))
```

---

### Q40. How do you check if a substring exists in a string?

**Answer:**

Use the **`in`** operator: **`sub in s`** returns a booleanтАФidiomatic and readable. Alternatives: **`s.find(sub)`** returns index or `-1`; **`s.index(sub)`** raises if missing; **`s.count(sub)`** for occurrences. For regex patterns, use **`re.search()`**.

Case-insensitive checks: normalize with **`lower()`** or use **`casefold()`** for Unicode-aware comparisons.

```python
text = "hello world"
print("world" in text)        # True
print(text.find("moon"))      # -1
```

---

### Q41. What is the difference between `str.find()` and `str.index()`?

**Answer:**

Both locate a substringтАЩs first occurrence. **`find`** returns **`-1`** if not foundтАФsafe when absence is normal. **`index`** raises **`ValueError`** if not foundтАФuse when absence is exceptional. Both accept `start` and `end` bounds like slicing.

Choose based on error-handling style: **`find`** for optional presence; **`index`** when you expect the substring to exist.

```python
s = "abc"
print(s.find("x"))   # -1
try:
    print(s.index("x"))
except ValueError as e:
    print("missing", e)
```

---

### Q42. How do you reverse a string in Python?

**Answer:**

Because strings are immutable sequences, use **slicing**: **`s[::-1]`** creates a reversed copy. Alternatively, **`reversed(s)`** returns an iterator of charactersтАФjoin with `''.join(reversed(s))`. Avoid in-place reversal (not possible on `str`).

For interviews, mention O(n) time and O(n) space for the new string.

```python
s = "Python"
print(s[::-1])
print("".join(reversed(s)))
```

---

### Q43. What is string immutability?

**Answer:**

**Immutability** means a `str` objectтАЩs value cannot change after creation. тАЬModifyingтАЭ methods return **new** strings; the original stays in memory until garbage-collected if unreferenced. This enables strings to be **hashable** (dict keys, set elements) and thread-safe for reads.

Repeated concatenation in a loop with `+=` can be inefficientтАФprefer **`''.join(iterable)`** for many pieces.

```python
s = "ab"
t = s + "c"   # new object; s still "ab"
```

---

### Q44. How do you convert a string to a list and vice versa?

**Answer:**

**String тЖТ list of characters**: `list(s)` splits into single-character strings. **`s.split()`** splits on whitespace; **`s.split(',')`** splits on delimiter. **List of strings тЖТ string**: **`''.join(chars)`** concatenates without separators; include delimiters as needed: **`', '.join(words)`**.

For bytes, use **`encode()`** / **`decode()`** with an encoding (UTF-8 default in many contexts).

```python
print(list("hi"))           # ['h', 'i']
print("a,b,c".split(","))   # ['a','b','c']
print("".join(['P', 'y']))  # Py
```

---

### Q45. What are escape characters in Python?

**Answer:**

Escape sequences start with **`\`** inside string literals: **`\n`** newline, **`\t`** tab, **`\\`** literal backslash, **`\'`** / **`\"`** quotes in strings, **`\uXXXX`** Unicode escapes. **Raw strings** `r"..."` disable most escapes except quote endings.

Use escapes when you need special characters in a single-line string; triple quotes often reduce the need for `\n`.

```python
print("line1\nline2")
print("C:\\Users\\name")
print("\u2665")  # тЩе
```

## Lists, Tuples & Sets

### Q46. What is a list in Python? How do you create one?

**Answer:**

A **`list`** is a **mutable**, ordered sequence. Create with **`[]`** literals or **`list(iterable)`**. Lists can hold heterogeneous types (though homogeneous lists are common). They support indexing, slicing, concatenation, and many in-place methods (`append`, `extend`, `sort`).

Lists are **dynamic arrays** in CPython (over-allocated for amortized append cost). For large numeric arrays, consider **`array.array`** or **NumPy**тАФoutside pure stdlib interviews but good to mention.

```python
nums = [1, 2, 3]
empty = []
from_range = list(range(3))
matrix = [[1, 2], [3, 4]]
```

---

### Q47. What are common list methods?

**Answer:**

**`append(x)`** adds one item at end; **`extend(iter)`** adds each element from an iterable. **`insert(i, x)`** shifts elements; **`remove(x)`** deletes first value `x`; **`pop(i)`** removes and returns at index (default last). **`clear()`** empties; **`sort()`** sorts in place; **`reverse()`** reverses in place. **`copy()`** shallow copy; **`count(x)`** occurrences; **`index(x)`** first index.

Know **`sort(key=..., reverse=True)`** for custom ordering.

```python
a = [3, 1, 2]
a.sort()
print(a)
a.append(4)
print(a.pop())
```

---

### Q48. What is the difference between `append()` and `extend()`?

**Answer:**

**`append(x)`** adds **`x` as a single element**, even if `x` is iterableтАФthe list grows by length 1. **`extend(iter)`** iterates and **adds each item** separatelyтАФthe list grows by `len(iter)` (for finite iterables).

**`append` + list** nests a list; **`extend`** flattens one level into the list.

```python
a = [1, 2]
a.append([3, 4])
print(a)  # [1, 2, [3, 4]]

b = [1, 2]
b.extend([3, 4])
print(b)  # [1, 2, 3, 4]
```

---

### Q49. How do you sort a list in Python?

**Answer:**

**`list.sort()`** sorts **in place** and returns `None` (watch chaining). **`sorted(iterable)`** returns a **new** sorted list, leaving the original unchanged. Both accept **`key`** (e.g., `key=len`, `key=str.lower`) and **`reverse=True`**.

For stable sorting, PythonтАЩs Timsort is stableтАФequal elements keep relative order. For dicts or objects, use **`key`** or **`operator.itemgetter`**.

```python
words = ["banana", "pie", "Washington"]
print(sorted(words, key=len))
nums = [3, 1, 2]
nums.sort()
print(nums)
```

---

### Q50. What is a tuple? How is it different from a list?

**Answer:**

A **`tuple`** is an **immutable**, ordered sequence, usually written **`(a, b)`**. Single-element tuples need a trailing comma: **`(1,)`**. Tuples are lighter-weight for fixed records and can be **hashable** if all elements are hashableтАФusable as dict keys, unlike lists.

Lists are for homogeneous sequences you mutate; tuples for **fixed collections** and **multiple return values**. Both support indexing and unpacking.

```python
point = (3, 4)
singleton = (1,)
print(point[0])
```

---

### Q51. What is tuple packing and unpacking?

**Answer:**

**Packing** combines values into a tuple: **`t = 1, 2, 3`**. **Unpacking** assigns elements to names: **`a, b, c = t`**. The right-hand side must match the number of targets (unless using **`*` star** capture: `a, *rest = seq`).

Unpacking works with any iterable in many contexts (for-loops, function returns). It enables elegant swaps: **`a, b = b, a`**.

```python
coords = 10, 20      # packing
x, y = coords        # unpacking
first, *mid, last = [1, 2, 3, 4, 5]
print(first, mid, last)
```

---

### Q52. What is a named tuple?

**Answer:**

**`collections.namedtuple`** (or **`typing.NamedTuple`** / **`dataclasses`**) creates tuple subclasses with **named fields** for readability (`p.x` instead of `p[0]`). They remain memory-efficient compared to full classes. Immutable like tuples.

`namedtuple` factories are lightweight **records**; for mutable records with defaults, **dataclasses** are often nicer in modern code.

```python
from collections import namedtuple
Point = namedtuple("Point", ["x", "y"])
p = Point(1, 2)
print(p.x, p.y)
```

---

### Q53. What is a set in Python?

**Answer:**

A **`set`** is an **unordered** collection of **unique**, **hashable** elements. Literals **`{1, 2, 3}`** (not `{}`тАФthatтАЩs a dict). **`set(iterable)`** builds from duplicatesтАФuniqueness enforced. Sets support fast membership **`in`** and set algebra (union, intersection).

**No indexing**тАФsets are not sequences. **`frozenset`** is an immutable set (hashable if elements are).

```python
s = {1, 2, 2, 3}
print(s)  # {1, 2, 3}
```

---

### Q54. What are common set operations (union, intersection, difference)?

**Answer:**

**Union** `|` or **`a.union(b)`**: elements in either set. **Intersection** `&` / **`intersection`**: both. **Difference** `-` / **`difference`**: in A not in B. **Symmetric difference** `^`: in either but not both. Methods often take **iterables**; operators require sets (use `set(iter)` otherwise).

These are fundamental in deduping, tag systems, and graph adjacency sometimes.

```python
a = {1, 2, 3}
b = {2, 3, 4}
print(a | b)  # {1,2,3,4}
print(a & b)  # {2,3}
print(a - b)  # {1}
```

### Q55. What is a frozenset?

**Answer:**

**`frozenset`** is an **immutable** set: no `add`/`remove`. Because it is immutable and contains only hashable items, a **`frozenset` itself can be hashable** and used as a dict key or set memberтАФregular sets cannot nest in sets.

Use when you need set semantics but immutability for hashing or API contracts.

```python
fs = frozenset([1, 2, 3])
print(fs & frozenset([2, 4]))
```

### Q56. How do you remove duplicates from a list?

**Answer:**

**Preserve order (Python 3.7+ dict preserves insertion order):** `list(dict.fromkeys(items))`. **Order not important:** `list(set(items))` (sets unordered). For unhashable items, manual loops or comprehension with seen-set of reprsтАФdepends on data.

**`sorted(set(items))`** if you want unique sorted output. Choose based on **order** and **hashability** requirements.

```python
items = [3, 1, 2, 1, 3]
print(list(dict.fromkeys(items)))  # order-preserving unique
print(sorted(set(items)))
```

### Q57. What is the difference between `remove()`, `pop()`, and `del`?

**Answer:**

**`list.remove(x)`** removes the **first** occurrence of **value** `x`; **`ValueError`** if missing. **`pop(i)`** removes by **index** and returns the item; default `-1` (last). **`del`** is a **statement** deleting names or slices: **`del lst[i]`**, **`del lst[i:j]`**тАФno return value.

Use **`remove`** when you know the value; **`pop`** when you need the removed item or stack/LIFO behavior; **`del`** for slices or deleting names.

```python
lst = [1, 2, 3, 2]
lst.remove(2)   # removes first 2
x = lst.pop()   # removes last
del lst[0]
```

## Dictionaries

### Q58. What is a dictionary in Python?

**Answer:**

A **`dict`** maps **hashable keys** to arbitrary values. Insertion-ordered since Python 3.7 (language guarantee 3.7+). Literals **`{"a": 1}`**; **`dict(a=1)`** for string keys; **`dict(zip(keys, vals))`**. Average **O(1)** lookup/insert/delete.

Keys must be immutable/hashable (`str`, `int`, `tuple` of immutables). Values can be any object. Common for JSON-like records, caches, and frequency counting.

```python
user = {"name": "Ada", "role": "dev"}
print(user["name"])
```

---

### Q59. How do you access, add, and modify dictionary elements?

**Answer:**

Access with **`d[key]`**тАФraises **`KeyError`** if missing. **`d.get(key, default)`** returns default safely. **Assignment** `d[key] = value` adds or updates. **`setdefault(key, default)`** sets if missing and returns value.

Iterate keys with **`for k in d`**, values with **`d.values()`**, pairs with **`d.items()`**.

```python
d = {}
d["x"] = 1       # add
d["x"] = 2       # modify
print(d.get("y", 0))
```

---

### Q60. What are common dictionary methods?

**Answer:**

**`get`**, **`setdefault`**, **`pop`**, **`popitem`** (LIFO since 3.7), **`clear`**, **`update`** (merge from mapping/iterable of pairs), **`keys`**, **`values`**, **`items`** (dynamic views), **`copy`** (shallow). **`fromkeys(iter, value=None)`** class method for default-valued keys.

**`collections.Counter`**, **`defaultdict`**, **`ChainMap`** extend patternsтАФworth mentioning for тАЬcommon in practice.тАЭ

```python
d = {"a": 1}
d.update({"b": 2})
print(d.pop("a"))
```

---

### Q61. What is the difference between `dict.get()` and `dict[key]`?

**Answer:**

**`d[k]`** always returns the value or raises **`KeyError`**тАФsignals programmer expectation that the key exists. **`d.get(k, default=None)`** returns **`default`** if missingтАФno exceptionтАФuse for **optional** keys.

For defaults that need computation only if missing, **`collections.defaultdict`** or **`setdefault`** helpтАФavoid calling expensive functions in `get`тАЩs default arg eagerly if using patterns incorrectly.

```python
d = {"a": 1}
print(d.get("b", 0))  # 0
try:
    print(d["b"])
except KeyError:
    print("missing")
```

---

### Q62. How do you iterate over a dictionary?

**Answer:**

**`for k in d:`** iterates **keys** (same as `d.keys()` in iteration). **`for v in d.values()`** for values. **`for k, v in d.items()`** for pairsтАФmost common. Do not mutate keys/size during iteration except via careful patterns (or iterate over a copy of keys).

Order follows insertion order (3.7+). For sorted iteration: **`sorted(d)`** on keys.

```python
d = {"b": 2, "a": 1}
for k in sorted(d):
    print(k, d[k])
```

---

### Q63. What is a nested dictionary?

**Answer:**

A **nested dictionary** stores dicts as values, building trees or tables like JSON objects. Access with chained brackets **`d["user"]["name"]`**тАФmay need **`get`** at each level to avoid errors, or use **`collections.defaultdict(dict)`**.

Deep copies require care: assignment shares inner dicts; use **`copy.deepcopy`** for independent clones.

```python
company = {"dept": {"name": "Eng", "size": 50}}
print(company["dept"]["name"])
```

---

### Q64. How do you merge two dictionaries?

**Answer:**

Python 3.9+: **`d1 | d2`** creates a new merged dict (later keys win). **`d1 |= d2`** updates `d1` in place. Older style: **`{**d1, **d2}`** unpacking or **`d1.update(d2)`** (mutates `d1`).

For nested merges, third-party or recursive helpers are neededтАФstdlib alone does shallow merge.

```python
a = {"x": 1}
b = {"y": 2, "x": 3}
print(a | b)
```

---

### Q65. What are dictionary comprehensions?

**Answer:**

Like list comprehensions but builds **`{key_expr: value_expr for ...}`**. Compact mapping from iterables, filtering with `if`. Keys must be uniqueтАФlater duplicates overwrite earlier in the comprehension execution order.

Readable one-liners for inversion, tallying, or transforming keysтАФavoid overly complex comprehensions without line breaks.

```python
squares = {n: n * n for n in range(5)}
even_squares = {n: n * n for n in range(5) if n % 2 == 0}
```

## Functions

### Q66. What is a function in Python? How do you define one?

**Answer:**

A **function** groups reusable logic under a name. Define with **`def name(parameters):`** followed by an indented block. **`return`** sends a value back; omitting it returns **`None`**. Functions are **objects**тАФcan be assigned, passed, stored in data structures.

**Annotations** (`def f(x: int) -> str`) document types; defaults follow parameters without defaults. **PEP 8** naming: `lowercase_with_underscores`.

```python
def add(a, b):
    return a + b

print(add(2, 3))
```

---

### Q67. What are arguments and parameters?

**Answer:**

**Parameters** are names in the **function definition**; **arguments** are **values passed** at call time. **Positional** arguments map left-to-right; **keyword** arguments `f(b=2, a=1)` map by name. After `*`, parameters are **keyword-only**; before `/` (PEP 570), **positional-only**.

**`*args`** collects extra positional args into a tuple; **`**kwargs`** collects extra keyword args into a dict.

```python
def greet(greeting, name="world"):
    return f"{greeting}, {name}!"

print(greet("Hi"))
print(greet("Hi", name="Ada"))
```

---

### Q68. What are default arguments?

**Answer:**

Default values bind at **function definition** time for the **object** referencedтАФ**mutable defaults** (`def f(x=[]):`) share one list across calls, a classic bug. Use **`None`** as default and assign inside: `def f(x=None): x = x or []` (care with falsy values) or `x = [] if x is None else x`.

Defaults are evaluated **once**, not per callтАФcritical interview topic.

```python
def bad(acc=[]):
    acc.append(1)
    return acc

print(bad())  # [1]
print(bad())  # [1, 1]  # shared list!
```

---

### Q69. What are *args and **kwargs?

**Answer:**

**`*args`** gathers excess **positional** arguments into a **tuple** named `args` (name conventional). **`**kwargs`** gathers excess **keyword** arguments into a **dict**. They enable wrappers and decorators forwarding arbitrary calls: **`func(*args, **kwargs)`**.

Ordering: normal params, `*args`, keyword-only params (if any), `**kwargs`. Unpacking at call site: **`f(*list, **dict)`** expands collections into arguments.

```python
def log(tag, *args, **kwargs):
    print(tag, args, kwargs)

log("INFO", 1, 2, x=3)
```

---

### Q70. What is the difference between `return` and `print`?

**Answer:**

**`return`** ends function execution and **passes a value to the caller**тАФthe expression becomes the functionтАЩs result. **`print`** writes human-readable text to **stdout** as a side effect and returns **`None`**.

Functions used in expressions or libraries should **`return`** data; **`print`** is for debugging or CLI output. Confusing them breaks composability (`result = f()` expects a value, not printed text).

```python
def square(x):
    return x * x

print(square(4))  # print shows 16; square returned 16
```

---

### Q71. What is a lambda function?

**Answer:**

A **lambda** is a small **anonymous** function: **`lambda params: expression`**. Limited to a **single expression** (no statements). Common in **`sorted(key=...)`**, **`map`**, **`filter`**. For readability, use a **`def`** if logic spans multiple steps.

Lambdas close over outer variablesтАФwatch loop variable binding in older patterns (use default args to capture).

```python
square = lambda x: x * x
print(sorted(["bb", "a", "ccc"], key=lambda s: len(s)))
```

---

### Q72. What are `map()`, `filter()`, and `reduce()`?

**Answer:**

**`map(func, iterable)`** applies `func` to each itemтАФreturns iterator; prefer **list comprehensions** often for clarity. **`filter(pred, iterable)`** yields items where `pred` is true. **`functools.reduce(func, iterable, init)`** cumulatively combines items (e.g., product, concat)тАФmust import **`reduce`** in Python 3.

All return iterators in Python 3 (`map`/`filter`); consume with `list()` if needed.

```python
from functools import reduce
print(list(map(lambda x: x * 2, [1, 2, 3])))
print(list(filter(lambda x: x > 1, [1, 2, 3])))
print(reduce(lambda a, b: a + b, [1, 2, 3], 0))
```

---

### Q73. What is variable scope (local, global, nonlocal)?

**Answer:**

**Scope** determines where names are visible. **Local** names are assigned inside a function (unless **`global`**/`nonlocal` declared). **Global** names are module-level. **Enclosing** function locals for nested defs. **`LEGB`** rule: Local, Enclosing, Global, Builtins lookup order.

**`global x`** rebinds module-level `x`. **`nonlocal y`** rebinds a name in an **enclosing** (non-global) functionтАФneeded for closures that assign to outer locals.

```python
count = 0

def inc():
    global count
    count += 1
```

---

### Q74. What is the `global` keyword?

**Answer:**

Assignment to a name inside a function **creates a local** unless declared **`global`**. **`global var`** means тАЬthis name refers to the **module-level** binding.тАЭ Read-only access to globals does not need `global`; assignment does.

Avoid excessive globalsтАФprefer parameters and return values for clarity and testing.

```python
x = 1

def f():
    global x
    x = 2

f()
print(x)
```

### Q75. What is recursion? Give an example.

**Answer:**

**Recursion** is when a function calls itself. Each call needs a **base case** to stop and typically **reduces** the problem size. Python has a **recursion depth limit** (default ~1000) to prevent stack overflowтАФ`sys.setrecursionlimit` exists but use carefully.

Classic examples: factorial, Fibonacci (inefficient na├пvely), tree traversals. Iterative versions often use less memory.

```python
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

print(factorial(5))
```

### Q76. What are docstrings?

**Answer:**

A **docstring** is a string literal as the **first statement** in a module, class, or function, conventionally triple-quoted. Tools extract it for **`help(obj)`**, IDEs, and Sphinx. **PEP 257** describes one-line vs multi-line styles.

Unlike comments, docstrings become **`__doc__`** attributes. First line should be a summary; extended description follows a blank line.

```python
def add(a, b):
    """Return the sum of a and b."""
    return a + b

print(add.__doc__)
```

### Q77. What are first-class functions?

**Answer:**

**First-class** means functions are **values**: assign to variables, pass as arguments, return from functions, store in structures. Enables **higher-order functions** (`map`, decorators) and **callbacks**.

Python functions are objects with attributes (`__name__`, `__doc__`). **Nested functions** and **closures** capture outer scope variables.

```python
def apply_twice(f, x):
    return f(f(x))

print(apply_twice(lambda y: y + 1, 0))
```

## Object-Oriented Programming Basics

### Q78. What is Object-Oriented Programming?

**Answer:**

**OOP** organizes code around **objects** combining **state** (attributes) and **behavior** (methods). Core ideas: **encapsulation** (bundling data with methods), **inheritance** (reuse/extend types), **polymorphism** (same interface, different implementations), and **abstraction** (hide complexity).

Python supports OOP but does not force itтАФmodules and functions are fine too. **Duck typing** complements nominal typing: тАЬif it walks like a duckтАжтАЭ

```python
class Duck:
    def quack(self):
        return "quack"

d = Duck()
print(d.quack())
```

---

### Q79. What is a class and an object?

**Answer:**

A **class** is a blueprint defining attributes and methods. An **object** (**instance**) is a concrete realization created by **`ClassName(...)`**, which calls **`__init__`** after **`__new__`**. Classes are objects too (instances of `type`).

**Instance attributes** live on `self`; **class attributes** are shared unless shadowed.

```python
class Counter:
    def __init__(self):
        self.n = 0

c = Counter()
print(type(c))
```

---

### Q80. What is the `__init__` method?

**Answer:**

**`__init__`** initializes a new instance after creation. It is **not** a constructor in the C++ senseтАФ**`__new__`** allocates; **`__init__`** configures `self`. It should return **`None`** (implicitly). Parameters after `self` define construction arguments.

If omitted, instances still get default behavior from `object`.

```python
class Person:
    def __init__(self, name):
        self.name = name

p = Person("Ada")
```

---

### Q81. What is `self` in Python?

**Answer:**

**`self`** is the conventional first parameter of instance methods referring to the **current object**. Python passes the instance automatically: **`obj.method()`** тЖФ **`Class.method(obj)`**. Name `self` is convention only, but **always use `self`** for readability.

**`cls`** is conventional for class methodsтАЩ first parameter. **`@staticmethod`** methods omit `self`/`cls`.

```python
class Box:
    def __init__(self, v):
        self.v = v

    def show(self):
        return self.v

print(Box(3).show())
```

---

### Q82. What are instance variables and class variables?

**Answer:**

**Instance variables** are attributes on **`self`**тАФunique per object unless explicitly shared. **Class variables** are defined in the class bodyтАФ**shared** across instances unless an instance assigns the same name (shadowing).

Mutable class variables as defaults are a common pitfall (shared list across instances).

```python
class Dog:
    species = "Canis"  # class variable

    def __init__(self, name):
        self.name = name  # instance variable
```

---

### Q83. What is inheritance? Give an example.

**Answer:**

**Inheritance** lets a **subclass** reuse and extend a **base class**. The subclass inherits methods unless overridden. **`isinstance(sub(), Base)`** is true. Python supports **multiple inheritance** with the **MRO** (Method Resolution Order) via **C3 linearization**.

Use inheritance for **is-a** relationships; favor **composition** when тАЬhas-aтАЭ fits better.

```python
class Animal:
    def speak(self):
        return "..."

class Dog(Animal):
    def speak(self):
        return "woof"

print(Dog().speak())
```

---

### Q84. What is method overriding?

**Answer:**

**Overriding** redefines a method in a subclass with the **same name** so dispatch selects the subclass version via polymorphism. Call **`super().method()`** to extend rather than replace base behavior.

Without overriding, the inherited implementation runs.

```python
class Base:
    def f(self):
        return 1

class Sub(Base):
    def f(self):
        return super().f() + 1

print(Sub().f())
```

---

### Q85. What is encapsulation in Python?

**Answer:**

**Encapsulation** hides internal state and exposes a controlled interface. Python uses **naming conventions**: leading `_` means тАЬinternal useтАЭ; **`__name`** triggers **name mangling** to reduce accidental subclass clashesтАФnot true privacy.

**Properties** (`@property`) provide getters/setters with attribute syntax. Culture favors тАЬweтАЩre all consenting adultsтАЭ vs strict access control.

```python
class Account:
    def __init__(self, balance):
        self._balance = balance

    @property
    def balance(self):
        return self._balance
```

---

### Q86. What is polymorphism?

**Answer:**

**Polymorphism** means **many forms**: different classes implement the same **interface** or method name, and callers work uniformly. PythonтАЩs duck typing emphasizes **behavior** over declared typesтАФif it has `read()`, treat it like a readable object.

**Operator overloading** via dunder methods (`__add__`) is another polymorphism form.

```python
class Cat:
    def speak(self):
        return "meow"

class Dog:
    def speak(self):
        return "woof"

for animal in [Cat(), Dog()]:
    print(animal.speak())
```

---

### Q87. What is the `super()` function?

**Answer:**

**`super()`** returns a proxy object for **delegating method calls** to parent/ sibling classes in the MROтАФused inside overridden methods to **extend** base behavior. In single inheritance, `super().method()` calls the parentтАЩs version.

In multiple inheritance, **`super()`** follows **MRO**, enabling cooperative patterns. **Zero-argument `super()`** inside methods (Python 3) binds to `self` and class automatically.

```python
class A:
    def __init__(self):
        self.a = 1

class B(A):
    def __init__(self):
        super().__init__()
        self.b = 2
```

## Modules & Packages

### Q88. What is a module in Python?

**Answer:**

A **module** is a file (`.py`) whose names become attributes when imported. It provides **namespace** separation and reuse. **`import math`** loads the standard **`math`** module; **`from math import sqrt`** binds `sqrt` locally.

**`__name__`** is `"__main__"` when run as a script, else the module nameтАФenables dual-use modules (Q90).

```python
# mymod.py
PI = 3.14

def area(r):
    return PI * r * r
```

---

### Q89. What is the difference between `import` and `from ... import`?

**Answer:**

**`import mod`** requires **`mod.name`** qualificationтАФclear namespace, avoids clashes. **`from mod import name`** binds `name` directlyтАФshorter calls but risk of shadowing and less obvious origin. **`from mod import *`** imports public names (`__all__` if defined)тАФdiscouraged in production (namespace pollution).

**`import mod as m`** aliases for brevity while keeping a prefix.

```python
import math
print(math.sqrt(9))

from math import sqrt
print(sqrt(9))
```

---

### Q90. What is `__name__ == "__main__"`?

**Answer:**

When a file is **executed** as the program entry (`python script.py`), **`__name__`** is set to **`"__main__"`**. When **imported**, **`__name__`** is the **module name** (e.g., `"mymod"`). The idiom **`if __name__ == "__main__":`** runs tests or a CLI only when the file is executed directly, not when imported.

This enables reusable modules that also act as scripts.

```python
def main():
    print("running")

if __name__ == "__main__":
    main()
```

**Answer:**

**pip** is PythonтАЩs **package installer** for **PyPI** (Python Package Index). Commands: **`pip install package`**, **`pip uninstall`**, **`pip list`**, **`pip freeze`** for requirements files. Use **virtual environments** (`python -m venv .venv`) to isolate dependencies per project.

**`pyproject.toml`** / **`requirements.txt`** document dependencies. **`pip`** is the standard tool in modern Python workflows (3.4+ ensured `ensurepip`).

## File Handling

### Q92. How do you read a file in Python?

**Answer:**

Open with **`open(path, mode)`**тАФtext mode default decodes bytes to `str` using an encoding (UTF-8 common). Read all: **`f.read()`**, line by line: **`for line in f`**, lines list: **`f.readlines()`**. Always prefer **`with open(...) as f:`** to ensure closure.

Specify **`encoding="utf-8"`** explicitly to avoid platform-dependent defaults.

```python
with open("data.txt", encoding="utf-8") as f:
    text = f.read()
```

---

### Q93. How do you write to a file?

**Answer:**

Open with **`"w"`** (truncate/create) or **`"a"`** (append). **`write(str)`** returns characters written; **`writelines(list)`** does not add newlinesтАФinclude `\n` yourself. **`print(..., file=f)`** is convenient.

Use **`with`** for automatic flush/close; **`flush()`** if immediate disk write needed.

```python
lines = ["a\n", "b\n"]
with open("out.txt", "w", encoding="utf-8") as f:
    f.writelines(lines)
```

---

### Q94. What is the `with` statement for file handling?

**Answer:**

**`with`** uses **context managers** (`__enter__`/`__exit__`) to acquire and **reliably release** resourcesтАФeven if exceptions occur. Files close automatically at block exit, preventing descriptor leaks.

Works beyond files: locks, DB connections, **`contextlib`** helpers. **`__exit__`** receives exception info and can suppress exceptions.

```python
with open("x.txt", encoding="utf-8") as f:
    data = f.read()
# file closed here even if read() raised
```

---

### Q95. What is the difference between `r`, `w`, `a`, and `r+` modes?

**Answer:**

**`r`** read-only (default text); fails if missing. **`w`** write: **truncates** existing file or creates new. **`a`** append: write at end; creates if missing. **`r+`** read/write without truncating; pointer at startтАФseek for append-like behavior.

Add **`b`** for binary (`rb`, `wb`)тАФbytes, no decoding. Text modes may translate newlines (`newline=` controls this).

```python
open("f.txt", "r", encoding="utf-8")
open("f.bin", "wb")
```

## Exception Handling

### Q96. What is exception handling in Python?

**Answer:**

An **exception** signals an error or exceptional condition, unwinding the stack until caught. **`try`/`except`** catches exceptions by type hierarchy; uncaught exceptions terminate the program with a traceback. This separates **error handling** from main logic compared to pervasive return codes.

**Built-in hierarchy** includes **`Exception`**, **`ValueError`**, **`TypeError`**, **`KeyError`**, etc. Define custom exceptions by subclassing **`Exception`**.

```python
try:
    int("x")
except ValueError as e:
    print("bad int", e)
```

---

### Q97. What is the difference between `try/except/else/finally`?

**Answer:**

**`try`**: run guarded code. **`except`**: handle matching exceptions (first matching type wins). **`else`**: runs if **`try` completed with no exception**тАФkeeps success path clear. **`finally`**: **always** runs last (even after `return`/`break`)тАФcleanup (close resources if not using `with`).

Order: `try` тЖТ (except if error) тЖТ else if no error тЖТ finally always.

```python
def demo():
    try:
        x = 1
    except RuntimeError:
        pass
    else:
        print("no error")
    finally:
        print("cleanup")

demo()
```

---

### Q98. How do you raise a custom exception?

**Answer:**

Define a class inheriting **`Exception`** (or a more specific built-in). **`raise MyError("message")`** triggers exception handling. Add attributes for context. **Chaining**: **`raise NewErr from old_err`** preserves cause for tracebacks.

Document which methods raise what for API clarity.

```python
class ValidationError(Exception):
    pass

def must_positive(x):
    if x <= 0:
        raise ValidationError("x must be positive")
    return x
```

## Comprehensions & Iterators

### Q99. What are list comprehensions?

**Answer:**

**List comprehensions** build lists concisely: **`[expr for item in iterable if condition]`**. They are often clearer and faster than `map`/`filter` for simple transforms. Nested comprehensions are possible but can harm readabilityтАФsometimes use loops or generator expressions.

**Iterators and iterables:** An **iterable** defines `__iter__()` and can be looped over; an **iterator** is an object with `__next__()` (and raises `StopIteration` when exhausted). `for` loops call `iter()` then `next()` under the hood. **Generator expressions** `(expr for ...)` are lazy iteratorsтАФmemory-friendly for large streams and can be passed to functions that consume iterators (`sum`, `max`, etc.).

```python
squares = [n * n for n in range(5)]
evens = [n for n in range(10) if n % 2 == 0]
total = sum(n * n for n in range(1000))  # generator expression, no list built
```
