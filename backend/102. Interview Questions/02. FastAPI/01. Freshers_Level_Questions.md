# FastAPI Interview Questions - Freshers Level (0-2 Years)

> 100 comprehensive questions and detailed answers covering FastAPI fundamentals for entry-level developers.

---

### Q1. What is FastAPI and why is it used?

**Answer:**

FastAPI is a modern, high-performance web framework for building APIs with Python 3.8+. It is built on top of Starlette (for the web parts) and Pydantic (for data validation). Teams choose FastAPI when they want **fast development**, **automatic OpenAPI documentation**, and **strong typing** with editor support. It is widely used for REST APIs, microservices, and backends that need async I/O. Because validation and serialization are declarative, you write less boilerplate and catch many bugs at request time instead of at runtime in random places.

```python
from fastapi import FastAPI

app = FastAPI(title="Orders API")

@app.get("/health")
def health():
    return {"status": "ok"}
```

---

### Q2. How does FastAPI compare to Flask and Django?

**Answer:**

**Flask** is a minimal WSGI framework: you add extensions for everything. It is flexible but does not ship with async-first design or automatic request/response schema docs out of the box. **Django** is a full-stack framework with ORM, admin, templates, and batteries included—great for large traditional web apps, heavier for small APIs. **FastAPI** focuses on APIs with **ASGI**, **Pydantic validation**, and **OpenAPI** generation by default. For pure JSON APIs and microservices, FastAPI often means faster iteration; for monolithic CMS-style apps, Django may still win. Flask sits in the middle for simplicity and ecosystem maturity.

---

### Q3. What are the key features of FastAPI?

**Answer:**

Key features include: **high performance** (comparable to Node/Go in many benchmarks, especially with async), **automatic interactive docs** (Swagger UI and ReDoc), **data validation** via Pydantic models, **dependency injection** for reusable logic, **first-class async** support, **WebSocket** support through Starlette, and **standards-based** design (OpenAPI, JSON Schema). It also gives excellent **IDE autocompletion** because routes and models use Python type hints. These features reduce glue code and make onboarding easier for teams that already use type hints.

```python
from fastapi import FastAPI
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float

app = FastAPI()

@app.post("/items/")
def create_item(item: Item):
    return item
```

### Q4. How to install FastAPI?

**Answer:**

Install FastAPI with **pip** (or **uv**, **poetry**, etc.). You almost always install an **ASGI server** such as **Uvicorn** alongside it for local development and production. Using a virtual environment is recommended so dependencies do not clash with system Python. For production, pin versions in `requirements.txt` or `pyproject.toml` for reproducible builds.

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install "fastapi[standard]" uvicorn
```

The `[standard]` extra pulls in common optional dependencies used in many tutorials (e.g., tooling around OpenAPI); you can also `pip install fastapi uvicorn` for a minimal setup.

### Q5. What is ASGI and how does FastAPI use it?

**Answer:**

**ASGI** (Asynchronous Server Gateway Interface) is a standard interface between Python async web apps and servers. Unlike WSGI, ASGI supports **async** request handlers, **WebSockets**, and long-lived connections. FastAPI applications are **ASGI apps**: the framework exposes an ASGI callable that servers like Uvicorn or Hypercorn invoke per connection. This lets you use `async def` endpoints and `await` I/O without blocking the event loop when written correctly.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/slow-io")
async def slow_io():
    # Example: await httpx.AsyncClient().get("https://example.com")
    return {"ok": True}
```

### Q6. What is Uvicorn and why is it needed?

**Answer:**

**Uvicorn** is an **ASGI server** implementation—it actually listens on a port, accepts HTTP connections, and passes them to your FastAPI app. FastAPI alone defines **what** to do with requests; Uvicorn (or another ASGI server) handles **how** connections are served. In development you run Uvicorn with `--reload` for auto-restart. In production you often place Uvicorn behind a reverse proxy (Nginx, Caddy) and may run multiple worker processes.

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Here `main:app` means “import `app` from module `main.py`.”

### Q7. How to create a basic FastAPI application?

**Answer:**

You instantiate `FastAPI()`, assign it to `app`, and register routes with decorators like `@app.get()`. The `app` object is the ASGI application you pass to Uvicorn. You can set metadata such as `title` and `version` for documentation. Keep `main.py` (or similar) small at first; split routers later as the project grows.

```python
from fastapi import FastAPI

app = FastAPI(title="Demo API", version="1.0.0")

@app.get("/")
def read_root():
    return {"message": "Hello World"}
```

### Q8. What is the role of the `@app.get()` decorator?

**Answer:**

`@app.get()` registers a **route** for the **HTTP GET** method on a given path. FastAPI uses the decorated function as the **handler**: it runs when a client issues GET to that path. The decorator can take a path string, and you can add parameters for OpenAPI metadata (`summary`, `tags`, etc.). Other decorators (`post`, `put`, `delete`, …) work the same way for their HTTP verbs.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

### Q9. How to run a FastAPI application?

**Answer:**

Run with an ASGI server—typically **Uvicorn**—pointing at your module and app object. Use `--reload` during development. Set `--host` to `0.0.0.0` when you need access from other machines on the network. In Docker or systemd, you configure the same command without `--reload` and tune workers for CPU cores.

```bash
uvicorn main:app --reload
```

You can also run programmatically (less common for beginners):

```python
import uvicorn

if __name__ == "__main__":
    uvicorn.run("main:app", host="127.0.0.1", port=8000, reload=True)
```

### Q10. What is automatic interactive documentation in FastAPI?

**Answer:**

FastAPI **generates OpenAPI** metadata from your routes, type hints, and Pydantic models. From that schema it serves **interactive documentation** so you can try endpoints in the browser without Postman. This reduces manual doc maintenance: when you change models, the docs update automatically. It is one of the biggest productivity wins for teams and interviewers often expect you to know where to find it (`/docs` and `/redoc` by default).

### Q11. What is Swagger UI and ReDoc in FastAPI?

**Answer:**

**Swagger UI** (`/docs` by default) is an interactive explorer where you can authenticate, fill parameters, and execute requests against your API. **ReDoc** (`/redoc` by default) is an alternative documentation layout—often nicer for reading long descriptions and schemas. Both are driven by the same OpenAPI schema FastAPI builds. You can disable or customize them via `FastAPI(docs_url=None, redoc_url=None)` if needed.

### Q12. What is the difference between ASGI and WSGI?

**Answer:**

**WSGI** is the older synchronous standard for Python web apps; each request is typically handled in a blocking style (though workers can parallelize). **ASGI** supports **async** and **multiple protocols** (HTTP, WebSockets). Frameworks like Flask/Django (classic) target WSGI; FastAPI targets ASGI. If you need async database drivers or many concurrent I/O-bound operations, ASGI is a natural fit. WSGI remains common for legacy apps and some hosting setups.

### Q13. What Python version does FastAPI require?

**Answer:**

FastAPI tracks modern Python releases. As of common FastAPI 0.100+ lines, **Python 3.8+** is typically required (always check `pyproject.toml` for the exact pin you install). Newer versions add better typing features. For interviews, saying “3.8+ with type hints” is usually enough; in real projects, prefer **3.10+** or **3.11+** for performance and syntax like `|` unions if you standardize on them.

### Q14. What is Starlette and how does FastAPI use it?

**Answer:**

**Starlette** is a lightweight ASGI toolkit: routing, requests/responses, middleware, WebSockets, background tasks, etc. **FastAPI builds on Starlette**—it inherits many behaviors (e.g., `Request`, `Response`, middleware stack) and adds Pydantic integration, dependency injection ergonomics, and automatic OpenAPI. Knowing Starlette helps when you read FastAPI docs about low-level request/response objects.

```python
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def add_header(request: Request, call_next):
    response = await call_next(request)
    response.headers["X-Framework"] = "FastAPI-on-Starlette"
    return response
```

### Q15. What is Pydantic and how does FastAPI use it?

**Answer:**

**Pydantic** validates and parses data using Python type annotations. FastAPI uses Pydantic models for **request bodies**, **response models**, and **settings** patterns. When a request arrives, FastAPI passes body/query/path data into Pydantic constructors; invalid data yields **422** validation errors with detailed JSON. This keeps validation logic declarative and centralized.

```python
from fastapi import FastAPI
from pydantic import BaseModel, Field

class Product(BaseModel):
    sku: str = Field(min_length=3)
    price: float = Field(gt=0)

app = FastAPI()

@app.post("/products/")
def create_product(p: Product):
    return {"received": p.model_dump()}
```

### Q16. How to define routes in FastAPI?

**Answer:**

Routes are functions registered on the `FastAPI` instance (or `APIRouter`) using decorators: `@app.get("/path")`, `@app.post`, etc. The **path string** can include `{param}` placeholders. You can also use `app.add_api_route` for imperative registration. For larger apps, split routes with `APIRouter` and `include_router` to keep modules organized.

```python
from fastapi import APIRouter, FastAPI

api = APIRouter(prefix="/v1")

@api.get("/items")
def list_items():
    return []

app = FastAPI()
app.include_router(api)
```

### Q17. What HTTP methods does FastAPI support?

**Answer:**

FastAPI supports standard HTTP verbs via decorators: **GET, POST, PUT, PATCH, DELETE, OPTIONS, HEAD**, and you can use generic routing APIs for less common cases. Each method maps to REST semantics: GET for reads, POST for creates, PUT/PATCH for updates, DELETE for deletes. OPTIONS/HEAD are often used by frameworks and clients automatically (CORS preflight uses OPTIONS).

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/x")
def get_x(): ...

@app.post("/x")
def post_x(): ...

@app.delete("/x")
def delete_x(): ...
```

### Q18. How to create a POST endpoint?

**Answer:**

Use `@app.post("/path")` and define parameters: **path**, **query**, **body** (often a Pydantic model), and **headers** as needed. POST handlers commonly accept JSON bodies; FastAPI parses JSON into your model automatically. Return a dict/list for JSON or set `response_model` to constrain output.

```python
from fastapi import FastAPI
from pydantic import BaseModel

class Login(BaseModel):
    username: str
    password: str

app = FastAPI()

@app.post("/login")
def login(data: Login):
    return {"token": "fake-token"}
```

### Q19. How to create PUT and DELETE endpoints?

**Answer:**

`@app.put` and `@app.delete` register handlers for those verbs. PUT is often used for **idempotent full replacement** of a resource; PATCH for partial updates (FastAPI treats them the same syntactically—you choose semantics). DELETE removes resources. Combine path parameters to identify the resource and return appropriate status codes (e.g., 204 No Content).

```python
from fastapi import FastAPI
from pydantic import BaseModel

class User(BaseModel):
    name: str

app = FastAPI()

@app.put("/users/{user_id}")
def replace_user(user_id: int, user: User):
    return {"user_id": user_id, **user.model_dump()}

@app.delete("/users/{user_id}")
def remove_user(user_id: int):
    return {"deleted": user_id}
```

### Q20. What is a path parameter in FastAPI?

**Answer:**

A **path parameter** is a dynamic segment in the URL path, declared with `{name}` in the route string and a matching function parameter. FastAPI **coerces types** from strings (e.g., `int`) and validates them. Path parameters are required by nature—they are part of the URL. Use them for resource identifiers.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}
```

---

### Q21. How to use multiple path parameters?

**Answer:**

Add multiple `{placeholders}` to the path and declare each as a function parameter with types. FastAPI matches names between path and parameters. Order in the function signature does not have to mirror URL order, but matching by **name** is what matters.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/orgs/{org_id}/projects/{project_id}")
def read_project(org_id: int, project_id: str):
    return {"org_id": org_id, "project_id": project_id}
```

### Q22. What are query parameters in FastAPI?

**Answer:**

**Query parameters** appear after `?` in the URL (`/search?q=hello&limit=10`). In FastAPI, any simple parameter not declared as path/body/header becomes a query parameter by default. They are great for filtering, pagination, and optional inputs. FastAPI reads them from the request query string and coerces types.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/search")
def search(q: str, limit: int = 10):
    return {"q": q, "limit": limit}
```

---

### Q23. How to set default values for query parameters?

**Answer:**

Give the parameter a **default in Python** (`limit: int = 20`). That makes it optional in the URL with a default if omitted. You can combine defaults with typing `Optional[...]` when you want `None` as a possible value. Defaults show up in OpenAPI as optional fields.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/items")
def list_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}
```

---

### Q24. How to make query parameters optional?

**Answer:**

Use `Optional[T] = None` or `T | None = None` (Python 3.10+) so the parameter can be absent. FastAPI will pass `None` when not provided. For strings, distinguish between empty string vs missing using Optional carefully. You can also use `Query(None)` for additional metadata and validation.

```python
from typing import Optional
from fastapi import FastAPI

app = FastAPI()

@app.get("/filter")
def filter_items(tag: Optional[str] = None):
    return {"tag": tag}
```

### Q25. What is the difference between path and query parameters?

**Answer:**

**Path parameters** belong to the resource identity in the REST path (`/users/5`). They are usually **required** and appear as segments. **Query parameters** modify the request (`/users?role=admin&page=2`) and are typically **optional** or repeated. Path params are best for IDs; query params for filters, sorting, and pagination. Both are validated and documented by FastAPI.

---

### Q26. How to handle multiple query parameters?

**Answer:**

Declare each as its own function parameter. For repeated keys (`tag=a&tag=b`), use `list` types (FastAPI collects them). You can group with `Query` for constraints. Avoid huge parameter lists—sometimes a Pydantic model with `Depends` is cleaner for many related queries.

```python
from fastapi import FastAPI, Query

app = FastAPI()

@app.get("/items")
def list_items(
    q: str | None = None,
    min_price: float | None = Query(default=None, ge=0),
    tags: list[str] = Query(default=[]),
):
    return {"q": q, "min_price": min_price, "tags": tags}
```

### Q27. What is the order of parameter resolution in FastAPI?

**Answer:**

FastAPI determines parameter type by **annotation** and **special classes**: parameters declared as Pydantic models (without `Body`/`Form` ambiguity) are treated as **body** for suitable methods; `Path`, `Query`, `Header`, `Cookie`, `Form`, `File`, `Depends`, `Security`, `Request`, `Response`, `WebSocket` are recognized. **Path params** match `{names}` in the route. The framework is **not** sensitive to Python parameter order for matching—it uses **names** and types. When ambiguous, you use explicit `Query()`, `Path()`, etc.

### Q28. How to use Enum for path parameters?

**Answer:**

Define a Python `Enum` and annotate the path parameter with it. FastAPI validates that only allowed values are accepted and documents the enum in OpenAPI. This is cleaner than stringly-typed routes with manual validation.

```python
from enum import Enum
from fastapi import FastAPI

class Color(str, Enum):
    red = "red"
    blue = "blue"

app = FastAPI()

@app.get("/colors/{color}")
def get_color(color: Color):
    return {"color": color.value}
```

### Q29. How to define path parameters with validation?

**Answer:**

Use `Path(...)` from FastAPI to attach constraints like `ge`, `le`, `min_length`, `regex`, title, description. This improves OpenAPI and returns **422** on bad input. You still declare the Python parameter next to `Path`.

```python
from fastapi import FastAPI, Path

app = FastAPI()

@app.get("/items/{item_id}")
def read_item(item_id: int = Path(ge=1, le=9999)):
    return {"item_id": item_id}
```

### Q30. How to handle path parameters with file paths?

**Answer:**

File paths may contain `/` which conflicts with routing. Use **`path` converters** in Starlette/FastAPI: declare the parameter type as `str` and use a path like `/files/{file_path:path}` so the segment captures slashes. Be careful: this can expose **path traversal** risks—sanitize and restrict to allowed directories.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/files/{file_path:path}")
def read_file(file_path: str):
    # Never blindly open(file_path) from user input in production
    return {"requested": file_path}
```

### Q31. How to receive a request body in FastAPI?

**Answer:**

For JSON bodies, declare a **Pydantic `BaseModel`** parameter on `POST`/`PUT`/`PATCH`. FastAPI reads JSON, validates, and instantiates the model. For multiple bodies or special cases, use `Body`. For form-encoded data, use `Form`/`File`. The body parameter should not conflict with path/query parameters with unclear types—when needed, be explicit.

```python
from fastapi import FastAPI
from pydantic import BaseModel

class Item(BaseModel):
    name: str

app = FastAPI()

@app.post("/items")
def create_item(item: Item):
    return item
```

### Q32. What is a Pydantic model in FastAPI?

**Answer:**

A **Pydantic model** is a class inheriting `BaseModel` with typed fields. It describes the **shape and constraints** of data. In FastAPI, models represent JSON request/response contracts. Validation errors become structured 422 responses. Pydantic v2 uses a Rust core (`pydantic-core`) for speed.

---

### Q33. How to define a Pydantic model?

**Answer:**

Subclass `pydantic.BaseModel`, declare fields with type annotations, optionally use `Field()` for constraints and metadata. You can add model-wide config via `model_config` in v2. Nested models compose larger structures.

```python
from pydantic import BaseModel, Field

class User(BaseModel):
    id: int
    email: str = Field(pattern=r"^[^@]+@[^@]+\.[^@]+$")
    age: int | None = None
```

---

### Q34. How to use nested Pydantic models?

**Answer:**

Use models as field types inside other models. FastAPI and Pydantic validate **recursively**. This maps cleanly to nested JSON objects (e.g., order with line items). Use lists of models for arrays of objects.

```python
from pydantic import BaseModel

class Address(BaseModel):
    city: str
    zip: str

class Person(BaseModel):
    name: str
    address: Address

# JSON: {"name": "A", "address": {"city": "X", "zip": "12345"}}
```

---

### Q35. How to set default values in Pydantic models?

**Answer:**

Assign defaults directly on fields (`status: str = "pending"`). For mutable defaults, use `Field(default_factory=list)` to avoid shared mutable state. Defaults affect both validation and OpenAPI examples.

```python
from pydantic import BaseModel, Field

class Task(BaseModel):
    title: str
    done: bool = False
    tags: list[str] = Field(default_factory=list)
```

---

### Q36. How to make fields optional in Pydantic models?

**Answer:**

Use `Optional[T]` or `T | None` and default to `None`, or use `Field(default=None)`. In Pydantic v2, required vs optional follows whether a default exists. Optional fields appear as not required in generated schemas.

```python
from pydantic import BaseModel

class Profile(BaseModel):
    name: str
    nickname: str | None = None
```

---

### Q37. What is Field() in Pydantic and how to use it?

**Answer:**

`Field()` attaches **validation constraints** and **schema metadata** (title, description, examples) to a model field. Examples: `gt`, `ge`, `min_length`, `max_length`, `pattern`. It replaces raw assignments when you need more than a plain default.

```python
from pydantic import BaseModel, Field

class Item(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    qty: int = Field(ge=1, le=999)
```

---

### Q38. How to add validation to Pydantic model fields?

**Answer:**

Use `Field` constraints, **type annotations**, and Pydantic **validators** (`field_validator`, `model_validator` in v2) for cross-field rules. This keeps business rules next to the schema. FastAPI automatically applies these validators on request bodies.

```python
from pydantic import BaseModel, model_validator

class User(BaseModel):
    password: str
    password2: str

    @model_validator(mode="after")
    def passwords_match(self):
        if self.password != self.password2:
            raise ValueError("passwords do not match")
        return self
```

### Q39. How to use multiple request body parameters?

**Answer:**

FastAPI normally expects **one** Pydantic body for JSON. If you need multiple distinct JSON bodies, you must use explicit `Body` parameters with `embed` or structure a **single** model. Multiple bodies are uncommon in REST; prefer one wrapper model. For forms, multiple `Form` parameters are normal.

```python
from fastapi import Body, FastAPI
from pydantic import BaseModel

class A(BaseModel):
    x: int

class B(BaseModel):
    y: int

app = FastAPI()

@app.post("/multi")
def multi(a: A = Body(), b: B = Body()):
    return {"a": a.model_dump(), "b": b.model_dump()}
```

### Q40. What is the difference between Body, Query, and Path?

**Answer:**

**Path** declares values from URL path segments; **Query** from query string; **Body** from JSON/form body content. They are FastAPI parameter classes used for extra validation and OpenAPI metadata when automatic inference is ambiguous. Using them explicitly makes APIs clearer in complex signatures.

```python
from fastapi import FastAPI, Path, Query, Body

app = FastAPI()

@app.put("/items/{item_id}")
def update_item(
    item_id: int = Path(ge=1),
    q: str | None = Query(default=None),
    payload: dict = Body(),
):
    return {"item_id": item_id, "q": q, "payload": payload}
```

### Q41. How does automatic data validation work in FastAPI?

**Answer:**

FastAPI uses **type hints + Pydantic**. For each request, it builds arguments: path/query/header/body. Pydantic parses strings to ints, validates constraints, and raises standardized errors. You rarely write manual `if not valid` checks for simple cases. The same mechanism powers OpenAPI schema generation.

### Q42. How to handle request body with path and query parameters together?

**Answer:**

Declare all three kinds in the same function: path parameters from `{id}`, query parameters as function args with defaults, and body as a Pydantic model for POST/PUT/PATCH. FastAPI merges them. Ensure names do not collide confusingly.

```python
from fastapi import FastAPI
from pydantic import BaseModel

class Patch(BaseModel):
    title: str | None = None

app = FastAPI()

@app.patch("/posts/{post_id}")
def patch_post(body: Patch, post_id: int, dry_run: bool = False):
    return {"post_id": post_id, "dry_run": dry_run, "body": body.model_dump()}
```

### Q43. What are Pydantic model methods like .model_dump()?

**Answer:**

In **Pydantic v2**, `model_dump()` exports the model to a **dict** (optionally with mode, include/exclude). Older docs mention `.dict()`—in v2 prefer `model_dump()`. Other helpers: `model_dump_json()`, `model_copy()`, `model_validate()` for parsing arbitrary objects. These are useful before returning data or logging.

```python
from pydantic import BaseModel

class M(BaseModel):
    a: int

m = M(a=1)
assert m.model_dump() == {"a": 1}
```

### Q44. How to use List and Dict types in request body?

**Answer:**

Annotate fields with `list[Item]` or `dict[str, int]` (Python 3.9+) or `typing.List/Dict`. Pydantic validates element types. For untyped JSON objects you can use `dict[str, Any]` but you lose strictness—prefer explicit models when possible.

```python
from pydantic import BaseModel

class Batch(BaseModel):
    ids: list[int]
    meta: dict[str, str]
```

### Q45. What is model_config in Pydantic v2?

**Answer:**

`model_config` is a **class attribute** on `BaseModel` replacing older `class Config`. It controls behaviors like **forbid extra fields**, validation strictness, serialization aliases, and more. FastAPI respects these settings when validating incoming bodies.

```python
from pydantic import BaseModel, ConfigDict

class Strict(BaseModel):
    model_config = ConfigDict(extra="forbid")
    name: str
```

### Q46. How to create example data for Pydantic models?

**Answer:**

Use `Field(examples=[...])`, `model_config` with `json_schema_extra`, or FastAPI `Body(example=...)`. Examples appear in OpenAPI and Swagger UI, helping consumers understand payloads.

```python
from pydantic import BaseModel, Field

class Item(BaseModel):
    name: str = Field(examples=["Widget"])
    price: float = Field(examples=[9.99])
```

### Q47. What is form data and how to handle it in FastAPI?

**Answer:**

**HTML forms** submit `application/x-www-form-urlencoded` or `multipart/form-data`. In FastAPI declare parameters with `Form()` instead of a JSON body model. You cannot mix unlimited JSON body with multiple forms the same way—use the right content type. For simple login forms, `OAuth2PasswordRequestForm` is a helper.

```python
from fastapi import FastAPI, Form

app = FastAPI()

@app.post("/login-form")
def login(username: str = Form(), password: str = Form()):
    return {"username": username}
```

### Q48. How to handle file uploads in FastAPI?

**Answer:**

Use `UploadFile` or `bytes` with `File()`. `UploadFile` streams to disk/spools and has `.filename` and async `.read()`. Declare content type `multipart/form-data` implicitly via `File`/`Form`. For large files, prefer streaming with `UploadFile`.

```python
from fastapi import FastAPI, File, UploadFile

app = FastAPI()

@app.post("/upload")
async def upload(file: UploadFile = File()):
    data = await file.read()
    return {"name": file.filename, "size": len(data)}
```

### Q49. How to handle multiple file uploads?

**Answer:**

### Q50. What is UploadFile vs bytes for file handling?

**Answer:**

**`bytes`** reads the entire upload into memory—simple but risky for large files. **`UploadFile`** is a thin wrapper around a spooled file object: async-friendly, can be read in chunks, exposes metadata. For production uploads, prefer `UploadFile` and enforce size limits (e.g., in middleware or by reading limited chunks).

```python
from fastapi import File, UploadFile

# bytes: small payloads only
async def as_bytes(data: bytes = File()): ...

# UploadFile: streaming
async def as_upload(file: UploadFile): ...
```

### Q53. What is response_model in FastAPI?

**Answer:**

`response_model` declares the **shape of the response** for OpenAPI and can **filter** fields from internal objects (e.g., hide passwords). FastAPI validates/serializes the returned value against that model when possible. It helps keep internal DB models separate from public API contracts.

```python
from fastapi import FastAPI

    name: str
    password: str

class UserOut(BaseModel):
    name: str

app = FastAPI()

@app.post("/users", response_model=UserOut)
def create_user(u: UserIn):
    return u  # password stripped by response_model
```

```python
from fastapi import FastAPI
```

### Q62. How to handle streaming responses?

### Q66. How to raise HTTP exceptions in FastAPI?

**Answer:**

Raise `HTTPException` with a status code and detail message. FastAPI converts it to a JSON error body with that status. Do not return error dicts manually for expected failures—raising keeps a consistent pattern.

```python
from fastapi import FastAPI, HTTPException

app = FastAPI()

@app.get("/items/{item_id}")
def get_item(item_id: int):
    if item_id < 0:
        raise HTTPException(status_code=400, detail="Invalid id")
    return {"item_id": item_id}
```

### Q67. What is HTTPException?

**Answer:**

`HTTPException` is a Starlette/FastAPI exception type carrying **HTTP status** and **detail** (string, dict, or list). When raised during a request, FastAPI renders a standard error response. It is meant for **expected** client errors (4xx) and some 5xx cases, not for arbitrary Python bugs.

### Q68. How to add custom error details?

```python
from fastapi import HTTPException

raise HTTPException(
```

### Q69. How to create custom exception handlers?

**Answer:**

Register with `@app.exception_handler(MyError)` async def handler(request, exc). Return `JSONResponse` with appropriate status. Use this for domain exceptions your services raise. Keep handlers small and log server-side details separately.

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
```

### Q70. How to handle validation errors?

```python
from fastapi import FastAPI
```

### Q71. What is RequestValidationError?

**Answer:**

### Q74. How to handle 500 internal server errors?

### Q75. What is the difference between HTTPException and generic Exception?

**Answer:**

`HTTPException` signals an **intentional** HTTP error with a known status—FastAPI treats it as a normal control-flow path for errors. A generic `Exception` is usually a **bug** or unexpected failure; you should log it and return 500 unless converted. Do not use generic exceptions for routine “not found” cases—use `HTTPException` or domain types with handlers.

### Q76. What is dependency injection in FastAPI?

**Answer:**

**Dependency injection** lets you declare reusable setup logic (DB sessions, auth, settings) as functions or classes and **inject** them into routes via `Depends()`. FastAPI resolves the dependency graph per request, caches within a request by default for the same dependency, and makes testing easier by swapping dependencies.

```python
from fastapi import Depends, FastAPI

def common_headers(x_request_id: str | None = None):
    return {"X-Request-Id": x_request_id}

app = FastAPI()

@app.get("/d")
def route(h=Depends(common_headers)):
    return h
```

---

### Q77. How to create a simple dependency?

**Answer:**

Write a function taking parameters FastAPI can resolve (query, headers) or other dependencies. Mark injection sites with `Depends(dep_func)`. Keep dependencies pure and fast; heavy work belongs in services with proper pooling.

```python
from fastapi import Depends, FastAPI

def pagination(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

app = FastAPI()

@app.get("/items")
def items(page=Depends(pagination)):
    return page
```

---

### Q78. How to use Depends()?

**Answer:**

`Depends(dependency)` wires a callable. FastAPI calls it with appropriate arguments. Nesting `Depends` builds trees. You can also `Depends()` with classes implementing `__call__` for callable class patterns.

```python
from fastapi import Depends, FastAPI

app = FastAPI()

def dep_a():
    return 1

def dep_b(a: int = Depends(dep_a)):
    return a + 1

@app.get("/z")
def z(v: int = Depends(dep_b)):
    return {"v": v}
```

---

### Q79. How to use class-based dependencies?

**Answer:**

Use a class with `__init__` taking injected parameters and methods for operations. FastAPI can treat the class as a dependency by depending on it directly (instantiated per request depending on signature) or use a callable pattern.

```python
from fastapi import Depends, FastAPI, Header

class Auth:
    def __init__(self, authorization: str | None = Header(None)):
        self.authorization = authorization

    def user(self) -> str | None:
        return self.authorization

app = FastAPI()

@app.get("/me")
def me(auth: Auth = Depends(Auth)):
    return {"auth": auth.user()}
```

### Q80. What are sub-dependencies?

**Answer:**

A dependency that itself declares `Depends(...)` forms a **chain**. FastAPI resolves inner dependencies first. This composes concerns (e.g., `get_db` used inside `get_repo`). Avoid deep chains without structure—refactor to services when logic grows.

```python
from fastapi import Depends, FastAPI

def token(header: str | None = None):
    return header

def user(tok: str | None = Depends(token)):
    return {"tok": tok}

app = FastAPI()

@app.get("/sub")
def route(u: dict = Depends(user)):
    return u
```

### Q81. How to share dependencies across routes?

**Answer:**

Put shared dependencies on **`APIRouter(dependencies=[Depends(...)])`** or individual routes. For app-wide needs, `FastAPI(dependencies=[...])`. This applies auth, DB session setup, etc., without repeating parameters everywhere.

```python
from fastapi import APIRouter, Depends, FastAPI

def require_token(): ...

router = APIRouter(dependencies=[Depends(require_token)])

@router.get("/secure")
def secure():
    return {"ok": True}

app = FastAPI()
app.include_router(router)
```

### Q82. What is the purpose of dependency injection in FastAPI?

**Answer:**

It reduces duplication, centralizes cross-cutting concerns (auth, transactions), improves **testability** (override dependencies in tests), and makes route functions smaller. Compared to global singletons, DI clarifies **per-request** lifetimes and scoping.

### Q83. How to use dependencies at router level?

**Answer:**

Pass `dependencies=` to `APIRouter` as shown above; all routes on that router inherit them. Combine router-level and route-level dependencies when some endpoints need extra checks.

### Q84. How to use yield dependencies?

**Answer:**

Dependencies can be **generators** with `yield` for setup/teardown (e.g., open DB connection, then close in `finally`). Code after `yield` runs after the request. This pairs well with database sessions.

```python
from fastapi import Depends, FastAPI

def get_db():
    db = {"open": True}
    try:
        yield db
    finally:
        db["open"] = False

app = FastAPI()

@app.get("/db")
def read_db(db: dict = Depends(get_db)):
    return db
```

---

### Q85. What is the dependency injection lifecycle?

**Answer:**

Dependencies are resolved **per request** (unless cached by FastAPI when the same dependency function is requested multiple times in one request—cached by default). `yield` dependencies run setup before the route and teardown after the response is ready. Background tasks may affect when teardown runs—understand ordering for transactions.

### Q86. What is middleware in FastAPI?

**Answer:**

**Middleware** wraps ASGI processing: it can inspect/modify requests and responses globally—logging, timing, adding headers, authentication probes. FastAPI exposes Starlette middleware API (`@app.middleware("http")` or `add_middleware`). Middleware runs for **every** request unless filtered internally.

```python
import time
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def add_timing(request: Request, call_next):
    start = time.perf_counter()
    response = await call_next(request)
    response.headers["X-Process-Time"] = f"{time.perf_counter() - start:.4f}s"
    return response
```

---

### Q87. How to create custom middleware?

**Answer:**

Use `@app.middleware("http")` async functions or subclass `BaseHTTPMiddleware` (less performant but convenient). Prefer pure ASGI middleware for hot paths when needed. Always call `await call_next(request)` unless short-circuiting.

### Q88. What is CORS and how to handle it in FastAPI?

**Answer:**

**CORS** (Cross-Origin Resource Sharing) is a browser security mechanism controlling which web origins may call your API with JavaScript. Browsers send **preflight OPTIONS** requests for non-simple requests. FastAPI handles this via **`CORSMiddleware`**, which adds proper `Access-Control-*` headers.

---

### Q89. How to add CORS middleware?

**Answer:**

`app.add_middleware(CORSMiddleware, allow_origins=[...], allow_methods, allow_headers, ...)`. In development people use `["*"]` but production should **whitelist** origins. Credentials require specific origins, not `*`.

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### Q90. What is TrustedHostMiddleware?

**Answer:**

It rejects requests whose `Host` header does not match allowed hosts, mitigating **Host header attacks** and cache poisoning in some setups. Configure `allowed_hosts` like Django’s `ALLOWED_HOSTS`.

```python
from fastapi import FastAPI
from starlette.middleware.trustedhost import TrustedHostMiddleware

app = FastAPI()
app.add_middleware(TrustedHostMiddleware, allowed_hosts=["example.com", "*.example.com"])
```

### Q91. How to add logging middleware?

**Answer:**

Log request method, path, status, duration in middleware. Use Python’s `logging` module, not `print`. Correlate with `X-Request-Id` from headers for tracing.

```python
import logging
from fastapi import FastAPI, Request

log = logging.getLogger("app")
app = FastAPI()

@app.middleware("http")
async def log_requests(request: Request, call_next):
    log.info("%s %s", request.method, request.url.path)
    response = await call_next(request)
    log.info("status=%s", response.status_code)
    return response
```

### Q92. What is the middleware execution order?

**Answer:**

Middleware forms a **stack**. The **last** `add_middleware` runs **first** on the incoming request (onion model). Order matters: CORS usually outer; compression inner. When debugging, log the order you register middleware.

### Q97. What is the lifespan event in FastAPI?

**Answer:**

```python
@app.get("/")
def root():
    return {"ok": True}
```
